# 브라우저가 폰트를 적용하는 과정 정리

## 핵심 요약

- 브라우저는 **HTML 파싱 → CSS 로드/파싱 → `@font-face` 확인 → 폰트 파일 로드(로컬/네트워크) → 렌더링** 흐름으로 폰트를 적용합니다.
- 폰트가 로드되기 전 텍스트 표시 방식은 `font-display` 값에 따라 달라지며, 대표적으로 **FOUT**(swap) / **FOIT**(block) 현상이 발생할 수 있습니다.
- **`preload`**(사전 로딩)과 **`font-display: optional`** 등을 통해 FOUT/FOIT를 완화할 수 있습니다.

## 브라우저 폰트 적용 흐름

### 1) HTML 문서 수신 및 파싱

사용자가 웹 페이지를 요청하면 **HTML 문서가 브라우저로 전송**됩니다. 브라우저는 HTML을 파싱하며 필요한 리소스를 수집합니다.

### 2) CSS 로드 및 파싱

브라우저는 HTML 파싱 과정에서 **CSS 파일을 로드**하고, CSS를 파싱하면서 **`@font-face` 선언 여부**를 확인합니다.

```css
@font-face {
  font-family: "MyWebFont";
  src: url("/fonts/myfont.woff2") format("woff2");
  font-weight: 400;
  font-style: normal;
  font-display: swap;
}
```

`@font-face`를 통해 브라우저는 **폰트 파일의 위치(src)와 스타일(굵기/기울임 등)** 정보를 파악합니다.

### 3) 폰트 로드 (로컬 또는 네트워크)

브라우저는 `@font-face`에 명시된 정보에 따라 폰트를:

- **로컬 시스템에서 가져오거나**
- **네트워크 요청으로 다운로드**합니다.

웹에서는 보통 용량과 압축률이 좋은 **`woff` / `woff2`** 형식이 널리 사용됩니다.

### 4) 폰트 로드 전/후 텍스트 렌더링과 `font-display`

폰트가 로드되기 이전에 텍스트가 어떻게 렌더링될지는 CSS의 `font-display` 설정에 따라 달라집니다.

## FOUT / FOIT 개념과 원인

### FOUT (Flash of Unstyled Text)

`font-display: swap`이면, **fallback 폰트로 먼저 렌더링**한 뒤 **웹 폰트가 로드되면 교체**됩니다.  
이때 텍스트 형태가 순간적으로 바뀌는 **FOUT 현상**이 발생할 수 있습니다.

```css
font-display: swap;
```

### FOIT (Flash of Invisible Text)

`font-display: block`이면, **웹 폰트가 완전히 로드될 때까지 텍스트가 보이지 않게** 됩니다.  
이로 인해 텍스트가 잠깐 숨었다가 나타나는(깜빡이는) **FOIT 현상**이 발생할 수 있습니다.

```css
font-display: block;
```

## FOUT / FOIT를 줄이는 방법

### 1) `preload`로 폰트 사전 로딩하기

사전 로딩을 위해 `preload`를 사용하면(예: `<link rel="preload" as="font">`) 브라우저가 **폰트 요청을 우선 처리**할 수 있습니다.  
그 결과 폰트를 더 이른 시점에 로드해 **FOUT/FOIT를 완화**할 수 있습니다.

```html
<link
  rel="preload"
  as="font"
  href="/fonts/myfont.woff2"
  type="font/woff2"
  crossorigin
/>
```

### 2) `font-display: optional` 사용하기

`font-display: optional`을 사용하면 폰트가 **빠르게 로드되지 않을 경우 fallback 폰트를 유지**함으로써 텍스트 형태가 순간적으로 변화하는 현상을 줄일 수 있습니다.

```css
font-display: optional;
```

## 장점

### `font-display: swap`

- 폰트가 늦게 로드되어도 **텍스트가 즉시 표시**됩니다(가독성/인지성 측면에서 유리).
- 사용자에게 “빈 화면” 느낌을 줄 가능성이 낮습니다.

### `preload`

- 폰트 다운로드 우선순위를 높여 **초기 렌더링 품질**을 개선할 수 있습니다.
- FOUT/FOIT가 발생할 시간을 줄여 사용자 경험을 개선합니다.

### `font-display: optional`

- 폰트가 느릴 때 **fallback을 유지**해 레이아웃/글꼴의 급격한 변화(플래시)를 줄일 수 있습니다.

## 단점

### `font-display: swap`

- 웹 폰트로 교체되는 순간 **텍스트 형태가 변하는 FOUT**이 나타날 수 있습니다.

### `font-display: block`

- 웹 폰트가 로드될 때까지 텍스트가 숨겨져 **FOIT(깜빡임/공백)**이 발생할 수 있습니다.

### `preload`

- 잘못/과도하게 사용하면 초기 네트워크 대역폭을 점유해 **다른 핵심 리소스 로딩에 영향**을 줄 수 있습니다.
- `crossorigin` 등 속성 설정을 누락하면 폰트 재요청이 발생하는 등 최적화 효과가 줄어들 수 있습니다.

## 주의사항 / 실무 팁

- 웹 폰트는 파일 크기와 요청 수가 성능에 직접 영향을 주므로, 가능한 한 **필요한 굵기/문자셋만 포함**하도록 설계합니다.
- `preload`를 사용할 때는 보통 `as="font"`, `type="font/woff2"`, `crossorigin` 등을 함께 고려합니다.
- `font-display` 선택은 제품 성격에 따라 달라질 수 있습니다.
  - 텍스트 즉시 노출이 중요하면 `swap`을,
  - 깜빡임보다 일관된 표시가 중요하면 `optional` 등을 검토합니다.
