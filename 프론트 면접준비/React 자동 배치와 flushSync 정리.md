# React 자동 배치와 flushSync 정리

## 핵심 요약
- `자동 배치(automatic batching)`는 여러 상태 업데이트를 React가 묶어서 한 번에 처리하는 방식이다.
- 목적은 불필요한 렌더링 횟수를 줄여 성능과 일관성을 높이는 것이다.
- `flushSync`는 이 배치를 깨고 즉시 반영이 필요할 때 강제로 동기 처리하는 도구다.
- 면접에서는 "기본은 배치를 믿고, 정말 필요한 경우에만 `flushSync`를 쓴다"라고 정리하면 좋다.

## 개념 설명
예전에는 이벤트 핸들러 안에서는 상태 업데이트가 묶였지만, 비동기 콜백에서는 그렇지 않은 경우가 있었다. React 18 이후에는 Promise, timeout 같은 비동기 경계에서도 상태 업데이트를 더 폭넓게 배치해서 처리한다.

이렇게 하면 `setState`를 여러 번 호출해도 매번 화면을 다시 그리지 않고, React가 적절한 시점에 한 번에 반영한다. 결과적으로 렌더링 횟수가 줄고 UI 상태도 더 예측 가능해진다.

반면 어떤 작업은 상태가 바로 DOM에 반영돼야 다음 로직이 안전하게 동작한다. 예를 들어 상태 변경 직후 특정 요소의 크기를 재거나 포커스를 이동해야 할 수 있다. 이럴 때 쓰는 것이 `flushSync`다.

## 예시
예를 들어 버튼 클릭 후 로딩 상태와 데이터 상태를 같이 바꾼다고 하자.

```tsx
setLoading(true);
setItems(nextItems);
```

자동 배치가 적용되면 React는 이 변경을 묶어서 한 번 렌더링할 수 있다.

반대로 모달을 연 직후 입력창에 즉시 포커스를 줘야 하는데, DOM 반영이 아직 안 된 상태라면 다음처럼 `flushSync`를 고려할 수 있다.

```tsx
flushSync(() => {
  setIsOpen(true);
});
inputRef.current?.focus();
```

## 면접 답변 예시
- 질문: "React 자동 배치는 무엇인가요?"  
  답변: "여러 상태 업데이트를 React가 한 번의 렌더링으로 묶어서 처리하는 방식입니다. React 18 이후에는 비동기 상황에서도 더 넓게 적용돼 불필요한 렌더링을 줄이는 데 도움이 됩니다."
- 질문: "`flushSync`는 언제 쓰나요?"  
  답변: "상태 변경 직후 DOM이 반드시 반영돼 있어야 하는 경우에만 제한적으로 사용합니다. 예를 들면 포커스 이동, 레이아웃 측정처럼 즉시성 요구가 분명한 경우입니다. 남용하면 React의 배치 이점을 잃을 수 있습니다."

## 장점
- 렌더링 횟수를 줄여 성능 개선에 도움이 된다
- 여러 상태 변경을 한 흐름으로 묶어 UI 일관성을 유지하기 좋다
- `flushSync`로 예외적인 즉시 반영 요구도 대응할 수 있다

## 단점
- 자동 배치를 이해하지 못하면 "왜 바로 안 바뀌지?" 같은 혼란이 생길 수 있다
- `flushSync`를 남용하면 성능 이점이 줄고 코드 의도가 흐려진다
- 상태 반영 시점을 DOM 조작과 강하게 결합하면 유지보수가 어려워질 수 있다

## 주의사항 / 실무 팁
- 기본 원칙은 React의 배치와 선언적 업데이트를 믿는 것이다.
- DOM 측정이나 포커스처럼 정말 즉시 반영이 필요한 경우에만 `flushSync`를 검토한다.
- 면접에서는 `setState`가 "즉시 변수처럼 바뀌는 것"이 아니라 React 스케줄링 안에서 반영된다는 점까지 같이 설명하면 깊이가 생긴다.
