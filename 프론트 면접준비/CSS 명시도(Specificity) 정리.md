# CSS 명시도(Specificity) 정리

## 핵심 요약

- CSS 명시도는 **스타일이 충돌할 때 어떤 규칙이 우선 적용되는지**를 결정하는 기준입니다.
- 선택자 종류에 따라 우선순위가 있으며, **점수 합산 방식**으로 승자가 결정됩니다.
- 점수가 같으면 **나중에 선언된 규칙**이 적용됩니다.
- `!important`는 명시도를 무시하고 우선순위를 높이지만, 과도 사용은 유지보수에 불리합니다.

## CSS 명시도란?

CSS 명시도는 스타일이 충돌할 때 어떤 것이 우선 적용될지를 결정하는 개념입니다.  
웹 페이지에서 여러 개의 CSS 규칙이 동일한 요소에 적용될 수 있는데, 이때 브라우저는 명시도를 계산하여 어떤 스타일을 적용할지 판단합니다.

## 명시도 우선순위

선택자의 종류에 따라 다음과 같은 우선순위를 가집니다.

1. 인라인 스타일
2. ID 선택자
3. 클래스, 가상 클래스, 속성 선택자
4. 요소 선택자

이 우선순위에 따라 점수를 부여해 합산하여 적용 규칙을 결정합니다.  
합산 점수가 동일한 경우, **나중에 선언된 규칙이 적용**됩니다.

## 예시

아래 예시에서는 `<p>` 요소에 요소/클래스/ID 선택자가 모두 적용될 수 있습니다.

```css
p {
  color: blue;
} /* 요소 선택자 */
.text {
  color: red;
} /* 클래스 선택자 */
#unique {
  color: green;
} /* ID 선택자 */
```

```html
<p id="unique" class="text">Hello, CSS!</p>
```

이 경우 명시도가 가장 높은 **ID 선택자(`#unique`)** 가 적용되어 글자가 초록색(green)으로 표시됩니다.

## 명시도가 높은 스타일을 강제로 덮어쓰려면?

`!important`를 사용하면 됩니다. `!important`는 스타일 규칙의 명시도를 무시하고 가장 높은 우선순위를 갖게 합니다.  
단, 동일한 속성에 대해 여러 개의 `!important` 규칙이 존재하면, 그때는 일반적인 명시도 규칙을 따릅니다.

```css
p {
  color: black !important;
}
```

## 장점

- 명시도 규칙을 이해하면, 스타일 충돌 상황에서 **어떤 규칙이 적용되는지 예측**하기 쉬워집니다.

## 단점

- `!important`를 남용하면 우선순위 역전이 잦아져 **유지보수성이 떨어질 수** 있습니다.

## 주의사항 / 실무 팁

- `!important`는 과도하게 사용하면 유지보수가 어려워질 수 있으므로 **필요한 경우에만** 사용하세요.
- 명시도 점수가 같은 상황에서는 “나중에 선언된 규칙이 승리”하므로, 스타일이 예상과 다르게 적용되면 **선언 순서**도 함께 점검하세요.
