# focus trap 구현에서 놓치기 쉬운 점 정리

## 핵심 요약

- 키보드 사용자가 모달 밖으로 빠지는 일을 막는다.
- 동적 컨텐츠가 바뀌면 오래된 포커스 목록이 틀어진다.
- 초기 포커스, 순환, 복귀를 별도 테스트로 둔다.

## 개념 설명

focus trap은 모달처럼 닫기 전까지 현재 영역 안에서만 키보드 포커스가 순환하도록 만드는 패턴이다.

열릴 때 첫 포커스 대상을 정하고, Tab과 Shift+Tab이 양 끝에서 순환하게 하며, 닫힌 뒤에는 열었던 요소로 포커스를 되돌린다.

## 예시

```js
const focusables = dialog.querySelectorAll("button, [href], input, select, textarea, [tabindex]:not([tabindex='-1'])");
first.focus();
```

포커스 가능한 요소 목록은 열릴 때마다 다시 계산해야 한다. 비활성 버튼이나 동적으로 추가된 필드를 포함하면 순환이 깨질 수 있다.

## 면접 답변 예시

> 대화상자 테스트 기준을 명확히 만든다. 브라우저 기본 dialog 동작과 직접 구현이 충돌할 수 있다. 모달 목적에 따라 제목, 첫 입력, 오류 요약 중 시작점을 고른다.

## 장점

- 닫기 후 원래 작업 위치를 회복하기 쉽다.

## 단점

- 초기 포커스를 무조건 첫 버튼에 두면 오류 메시지를 놓칠 수 있다.

## 주의사항 / 실무 팁

- 비활성 요소와 `tabindex=-1`을 필터링한다.
