# Core Web Vitals 정리

## 핵심 요약

Core Web Vitals는 구글이 웹사이트 **사용자 경험(UX)**을 평가하기 위해 제안한 핵심 지표로, **UX뿐 아니라 SEO에도 영향**을 줄 수 있어 프론트엔드에서 중요합니다. 대표 지표는 **LCP, INP, CLS** 3가지입니다.

## 개념

- Core Web Vitals는 웹 페이지의 체감 성능과 안정성을 수치로 평가하기 위한 지표 모음입니다.
- 구성 요소
  - **LCP (Largest Contentful Paint)**
  - **INP (Interaction to Next Paint)**
  - **CLS (Cumulative Layout Shift)**

## 지표 설명

### LCP (Largest Contentful Paint)

- **의미**: 페이지에서 **가장 큰 콘텐츠 요소**가 로드되는 시간을 측정
- **관점**: “주요 콘텐츠가 얼마나 빨리 보이는가”
- **좋음 기준**: 일반적으로 **2.5초 이내**

### INP (Interaction to Next Paint)

- **의미**: 사용자가 페이지를 이용하는 전체 기간 동안의 **상호작용 응답성**을 평가
- **측정 방식(요지)**: 클릭/탭/키보드 상호작용 지연을 관찰하고, **관찰된 가장 긴 상호작용**을 기준으로 최종 INP를 산정
- **좋음 기준**: **200ms 이하**

### CLS (Cumulative Layout Shift)

- **의미**: 페이지가 **예상치 못하게 시각적으로 이동**하는 정도를 측정
- **사례**: 읽던 문장이 갑자기 밀리거나 버튼 위치가 바뀌어 오클릭이 발생
- **좋음 기준**: **0.1 이하**

## 각 지표를 어떻게 개선할 수 있나요?

### LCP 개선

- 이미지 및 폰트 최적화
- 서버 응답 시간 단축

### INP 개선

- 불필요한 자바스크립트 실행 줄이기
- 코드 분할(Code Splitting)로 메인 스레드 부담 낮추기

### CLS 개선

- 이미지 크기를 명시적으로 설정
- 웹 폰트 로드 전략 조정

## 장점

- UX를 구성하는 중요한 요소를 **명확한 지표로 관리**할 수 있음
- 최적화 포인트가 비교적 분명해 **개선 활동의 우선순위**를 잡기 쉬움
- UX와 연계된 요소로서 **SEO에도 간접/직접적으로 유리**할 수 있음

## 단점

- 지표만 맞추는 최적화에 집중하면, 서비스 맥락상 중요한 사용자 경험을 놓칠 수 있음
- 실제 개선에는 이미지/폰트/JS/서버 등 **다양한 영역의 협업**이 필요할 수 있음

## 주의사항 및 실무 팁

- **지표별 병목 지점을 분리**해서 접근하세요. (로딩=LCP, 응답성=INP, 안정성=CLS)
- CLS 개선 시:
  - 레이아웃에 영향을 주는 요소(이미지/광고/폰트)를 “나중에 끼워 넣기”보다 **처음부터 공간을 확보**하는 방식이 유리합니다.
- INP 개선 시:
  - 메인 스레드를 점유하는 작업(무거운 JS, 긴 작업)을 줄이고 **작업을 쪼개거나 지연/분리**하는 전략을 고려합니다.
- LCP 개선 시:
  - “가장 큰 콘텐츠”가 무엇인지 파악한 뒤, 해당 리소스의 **다운로드/렌더링 경로**부터 최적화합니다.
