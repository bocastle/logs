# Import Maps 정리

## 핵심 요약
- `Import Maps`는 브라우저가 ES Module import 경로를 해석하는 방식을 개발자가 직접 매핑할 수 있게 해 주는 기능이다.
- 번들러 없이도 `react`, `lodash` 같은 모듈 이름을 원하는 URL이나 경로로 연결할 수 있다는 점이 핵심이다.
- 면접에서는 "브라우저 환경에서 모듈 경로 별칭을 표준 방식으로 제공하는 장치"라고 설명하면 전달이 빠르다.
- 다만 생태계가 아직 번들러 중심이기 때문에, 실무에서는 적용 범위와 브라우저 지원 전략을 함께 말하는 것이 좋다.

## 개념 설명
기본적으로 브라우저의 ES Module은 `./utils.js`, `/scripts/app.js`, `https://cdn.example.com/lib.js`처럼 실제 위치가 드러나는 경로를 요구한다. 반면 Node.js나 번들러 환경에서는 `import React from "react"`처럼 추상화된 이름을 자연스럽게 쓴다. 이 차이 때문에 브라우저만으로 모듈을 직접 로딩하려고 하면 경로 관리가 금방 복잡해진다.

`Import Maps`는 이런 문제를 줄이기 위해, HTML 안에서 "이 모듈 이름은 이 URL로 해석하라"는 규칙을 먼저 선언하는 방식이다. 그러면 브라우저는 이후 `import "react"` 같은 구문을 만났을 때 import map을 참고해서 실제 파일 위치를 찾는다.

핵심은 두 가지다.

- 모듈 이름과 실제 파일 위치를 분리할 수 있다
- 브라우저 표준 기능으로 경로 별칭을 줄 수 있다

즉, 예전에는 번들러 설정 파일에서 하던 별칭 관리 일부를 브라우저 수준으로 끌어온 개념이라고 볼 수 있다. 특히 작은 앱, 문서 사이트, 실험용 프로젝트, 마이크로 프론트엔드 일부 구간에서는 구조를 단순하게 만드는 데 도움이 된다.

## 예시
예를 들어 브라우저에서 다음처럼 모듈을 불러오고 싶다고 해 보자.

- `import { h } from "preact"`
- `import { render } from "preact/render-to-string"`

브라우저는 원래 `"preact"`가 어디를 가리키는지 모른다. 이때 import map으로 `"preact"`와 `"preact/"`를 CDN URL에 연결해 두면, 애플리케이션 코드는 패키지 이름 중심으로 더 읽기 쉬워진다.

또 다른 예시는 사내 공통 유틸 모듈이다. 여러 페이지에서 `/assets/shared/date.js`를 직접 쓰는 대신 `"@shared/date"` 같은 별칭으로 통일하면 import 구문이 덜 지저분해지고, 파일 위치가 바뀌어도 map만 수정하면 된다.

## 면접 답변 예시
- 질문: "`Import Maps`는 무엇인가요?"  
  답변: "브라우저에서 ES Module import 경로를 해석하는 규칙을 미리 선언하는 표준 기능입니다. 예를 들어 `react` 같은 이름을 특정 CDN URL이나 내부 경로로 매핑해서 번들러 없이도 모듈 별칭처럼 사용할 수 있습니다."
- 질문: "언제 유용한가요?"  
  답변: "작은 규모의 웹앱이나 문서 사이트처럼 번들링 복잡도를 줄이고 싶을 때 유용합니다. 또 브라우저에서 직접 모듈을 불러오는 구조에서 경로 관리 일관성을 높이는 데 도움이 됩니다."

## 장점
- 브라우저 표준 방식으로 모듈 별칭을 줄 수 있어 개념이 명확하다
- 번들러 의존도를 낮춘 간단한 앱 구조를 만들기 좋다
- CDN 기반 모듈 로딩이나 마이크로 프론트엔드 실험에 설명 포인트가 있다

## 단점
- 실무 생태계는 여전히 번들러와 패키지 매니저 중심이라 범용성이 제한될 수 있다
- 브라우저 지원, 캐싱 전략, 배포 구조를 함께 고려해야 한다
- 복잡한 대규모 앱에서는 번들 최적화 문제까지 해결해 주는 것은 아니다

## 주의사항 / 실무 팁
- `Import Maps`는 경로 해석 규칙을 바꾸는 기능이지, Tree Shaking이나 코드 분할을 자동으로 해결해 주는 기능은 아니다.
- 브라우저 호환성 정책이 민감한 서비스라면 폴리필이나 대체 전략을 같이 검토해야 한다.
- 면접에서는 "`번들러의 역할 전체를 대체한다`가 아니라 `브라우저에서 모듈 경로 별칭을 표준화한다`"는 정도로 설명하는 편이 정확하다.
- 실무에서는 소규모 페이지, 디자인 시스템 문서, 내부 어드민 도구처럼 구조 단순성이 중요한 곳에 먼저 적용해 보는 접근이 현실적이다.
