# 폼 자동완성 autocomplete 속성을 설계하는 기준 정리

## 핵심 요약

- `autocomplete`은 브라우저가 입력 필드의 의미를 이해하고 저장된 값을 제안할 수 있게 돕는 HTML 속성이다.
- 면접에서는 "편의 기능"뿐 아니라 접근성, 모바일 입력 경험, 보안 민감도까지 같이 설명하면 좋다.
- 이름, 이메일, 주소처럼 반복 입력되는 값은 돕고, 일회성 인증 코드나 민감한 값은 의도에 맞게 제한해야 한다.

## 개념 설명

폼은 사용자가 직접 시간을 쓰는 구간이다. 이메일, 주소, 전화번호처럼 매번 비슷하게 입력하는 값은 브라우저 자동완성이 잘 동작하면 전환율과 사용성이 좋아진다. 반대로 필드 이름이 애매하거나 `autocomplete`이 빠져 있으면 브라우저가 잘못된 값을 제안하거나, 모바일에서 적절한 키보드가 뜨지 않을 수 있다.

`autocomplete="email"`, `autocomplete="name"`, `autocomplete="one-time-code"`처럼 필드 목적을 명확히 적으면 브라우저와 보조 기술이 입력의 의미를 더 잘 이해한다. 특히 로그인, 결제, 주소 입력처럼 사용자가 실수하기 쉬운 폼에서는 작은 차이가 꽤 크게 느껴진다.

다만 자동완성을 무조건 켜는 것이 정답은 아니다. 공용 기기, 관리자 화면, 민감 정보 입력처럼 저장되면 곤란한 값은 보안 요구사항과 함께 판단해야 한다.

## 예시

```html
<form>
  <label>
    이메일
    <input type="email" name="email" autocomplete="email" />
  </label>

  <label>
    인증 코드
    <input
      type="text"
      name="code"
      inputmode="numeric"
      autocomplete="one-time-code"
    />
  </label>
</form>
```

자동완성 속성은 `name`이나 `type`을 대체하지 않는다. 셋이 함께 맞아야 브라우저가 더 안정적으로 해석한다.

## 면접 답변 예시

> `autocomplete`은 브라우저에게 이 입력 필드가 어떤 의미인지 알려 주는 속성입니다. 이메일, 이름, 주소처럼 반복 입력되는 값에는 정확한 토큰을 넣어 자동완성과 모바일 입력 경험을 돕고, 인증 코드에는 `one-time-code`처럼 목적에 맞는 값을 사용할 수 있습니다. 다만 민감 정보나 공용 기기에서 문제가 될 수 있는 값은 보안 요구사항을 같이 봐야 합니다. 저는 폼을 만들 때 label, name, type, autocomplete을 함께 맞추고 실제 브라우저와 모바일에서 자동완성이 어떻게 뜨는지 확인하겠다고 설명하겠습니다.

## 장점

- 사용자가 반복 입력하는 시간을 줄일 수 있다.
- 모바일에서 더 적절한 입력 제안과 키보드 경험을 제공할 수 있다.
- 보조 기술이 필드 의미를 파악하는 데 도움을 줄 수 있다.

## 단점

- 잘못된 토큰을 넣으면 엉뚱한 값이 제안될 수 있다.
- 브라우저마다 자동완성 동작이 조금씩 다를 수 있다.
- 민감한 정보는 자동 저장이 보안 이슈가 될 수 있다.

## 주의사항 / 실무 팁

- `label`, `name`, `type`, `autocomplete`을 서로 맞춰야 한다.
- `autocomplete="off"`는 브라우저가 항상 그대로 따르지 않을 수 있으므로 보안 통제로만 믿으면 안 된다.
- 로그인, 결제, 주소 입력 폼은 데스크톱과 모바일에서 실제 자동완성 동작을 확인하는 것이 좋다.
