# CSS Houdini Paint API 정리

## 핵심 요약

- CSS Houdini의 `paint()` API는 개발자가 JavaScript로 사용자 정의 배경, 테두리, 그래픽을 렌더링할 수 있게 해준다.
- 핵심 가치는 성능보다도 컴포넌트 기반 디자인 시스템에서 일관된 시각 규칙을 CSS 생태계 안에서 표현할 수 있다는 점이다.
- 면접에서는 보통 사용 시기, 브라우저 지원 범위, 폴백 전략을 묻는 질문에 대비해야 한다.

## 개념 설명

### 1) 무엇을 하는 기능인가

Paint API는 JavaScript로 캔버스를 그린 뒤, 그 결과를 CSS 값으로 노출해 스타일링에서 재사용한다.

```css
.my-box {
  --bg: paint(myGrid);
  background: paint(myGrid);
  background-size: 16px 16px;
}
```

```js
CSS.paintWorklet.addModule('/paint-worklet.js');
```

### 2) 언제 쓰는가

- 테마가 자주 바뀌고, 브랜딩 패턴이 반복되는 서비스
- 동일한 그래픽 규칙을 수십 개 컴포넌트에서 공유해야 하는 경우

### 3) 한계

- 지원 브라우저 점검 필요
- 복잡한 연산을 과도하게 넣으면 렌더링 비용 증가

## 사용 예시

- 배경 패턴(도트, 그리드)
- 미세한 보더 디테일(점선, 구획선)
- 데이터 시각화용 간단 아이콘 패턴

## 면접 답변 예시

"Houdini Paint API는 CSS의 표현력을 확장해 JS로 커스텀 렌더링을 가능하게 합니다. 단, 브라우저 지원과 캐시 전략을 함께 고려해야 합니다. 대규모 화면에서 연산량이 커지면 성능 회귀가 생길 수 있어, 단순 패턴 계열부터 적용해 점진적으로 확장하는 것이 안전합니다."

## 장점

- 디자인 시스템 규칙을 스타일 단계로 통합
- 반복 UI에서 일관성 개선
- 기존 CSS 구조 안에서 구현 가능

## 단점

- 학습 비용
- 브라우저 호환성 점검 필요
- 복잡도 증가 시 디버깅 난이도 상승

## 주의사항 / 실무 팁

- `paint` 모듈은 배포 시 캐시 버전 관리가 중요하다.
- 폴백 CSS(`background-color`/기본 패턴)으로 대체 경로를 꼭 둔다.
- 배치 전 핵심 Web Vitals 영향 지표를 꼭 확인한다.
