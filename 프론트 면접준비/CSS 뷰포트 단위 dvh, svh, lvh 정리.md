# CSS 뷰포트 단위 dvh, svh, lvh 정리

## 핵심 요약

- `dvh`, `svh`, `lvh`는 **모바일 브라우저에서 주소창/네비게이션 바 등 UI 변화로 인해 `vh`가 흔들리는 문제**를 보완하기 위해 나온 **뷰포트 높이(Height) 기준 단위**입니다.
- `dvh`: UI 변화에 따라 **실시간으로 변하는(동적) 뷰포트 높이**
- `svh`: 뷰포트가 **가장 작을 때(최소 높이)** 기준
- `lvh`: 뷰포트가 **가장 클 때(최대 높이)** 기준

## 배경: 왜 vh만으로는 부족했나

기존 `vh(viewport height)`는 브라우저의 뷰포트 높이를 기준으로 하지만, 모바일 환경에서는 **주소창/네비게이션 바의 표시 여부**에 따라 뷰포트 높이가 동적으로 변할 수 있습니다.  
이 때문에 “100vh로 꽉 채웠는데 스크롤/주소창 토글 시 레이아웃이 튄다” 같은 문제가 발생할 수 있고, 이를 보완하기 위해 `dvh`, `svh`, `lvh`가 등장했습니다.

## dvh, svh, lvh 정의와 동작

### dvh (Dynamic Viewport Height)

- **동적인 뷰포트 높이**를 의미합니다.
- 사용자가 스크롤하거나 모바일 주소창 같은 UI 요소가 나타나거나 사라질 때마다 **변하는 뷰포트 높이를 즉시 반영**합니다.
- 예: 모바일에서 주소창이 사라져 뷰포트가 커지면, 그 증가분을 **실시간 반영**한 값이 `dvh`입니다.

### svh (Small Viewport Height)

- 뷰포트가 **가장 작을 때의 높이(최소 높이)**를 의미합니다.
- 일반적으로 모바일 브라우저에서 **주소창이 보이는 상태의 최소 높이**를 기준으로 합니다.
- `svh`를 사용하면 화면 공간이 **가장 제한되는 시나리오**에서도 UI가 깨지지 않도록 레이아웃을 설계할 수 있습니다.

### lvh (Large Viewport Height)

- 뷰포트가 **가장 클 때의 높이(최대 높이)**를 의미합니다.
- 모바일에서 주소창이 사라진 뒤 **전체 화면이 확장된 상태의 최대 뷰포트 높이**를 나타냅니다.
- 화면을 최대한 활용하는 UI 구성에 사용할 수 있습니다.

## dvh가 더 유용해 보이는데, svh가 꼭 필요한 경우

`dvh`는 가변적이라 자연스러운 경험을 제공할 수 있지만, `svh`가 필요한 특수한 상황도 있습니다.

### 1) 빠른 스크롤/주소창 토글이 잦아 레이아웃이 “흔들려 보이는” 경우

- `dvh`는 주소창이 움직일 때마다 높이를 재계산하므로, 레이아웃이 **미세하게 흔들려 보일 수** 있습니다.
- `svh`는 최소 높이로 **고정**되어 있어 이런 떨림 현상이 덜합니다.

### 2) 성능이 중요한 환경(특히 모바일)

- `dvh`는 지속적으로 변하기 때문에 요소의 레이아웃 변화를 유발하여 **리플로우(reflow)를 자주 발생**시킬 수 있습니다.
- 모바일처럼 성능 차이가 체감되는 환경에서 **안정적인 UX**를 위해 `svh`가 더 적합할 수 있습니다.

## 장점

- 모바일 UI(주소창/네비게이션 바) 변화에 따른 `vh`의 불안정성을 보완해, **의도한 높이 계산**을 더 정확하게 할 수 있습니다.
- 상황에 따라 최소/최대/동적 뷰포트를 선택해 **레이아웃 안정성**과 **화면 활용도**를 조절할 수 있습니다.

## 단점

- `dvh`는 UI 변화에 따라 값이 계속 바뀌어, 상황에 따라 **레이아웃이 미세하게 튀거나 흔들려 보일 수** 있습니다.
- `dvh` 사용 시 잦은 레이아웃 변화로 **리플로우가 증가**할 수 있어 성능 이슈를 체감할 수 있습니다.

## 주의사항 / 실무 팁

- “항상 100% 꽉 차게”가 최우선이면 `dvh`가 직관적이지만, **화면 흔들림이 거슬리거나 성능이 민감**하면 `svh`가 더 안정적일 수 있습니다.
- **최소 높이 기준으로 UI 깨짐을 방지**해야 하는 경우(주소창이 있는 상태에서도 반드시 레이아웃이 유지되어야 함)에는 `svh`를 우선 고려하세요.
- 전체 화면을 적극 활용하는 UI(주소창이 사라진 상태에서의 최대 영역을 기준)에는 `lvh`가 도움이 될 수 있습니다.

## 예시 코드

### 1) 전체 화면 섹션을 “동적으로” 채우기 (dvh)

```css
.fullscreen {
  height: 100dvh;
}
```

### 2) 주소창이 보이는 “최소 높이”에서도 안정적으로 보이게 (svh)

```css
.safe-screen {
  min-height: 100svh;
}
```

### 3) 주소창이 사라진 “최대 높이”를 적극 활용 (lvh)

```css
.max-screen {
  height: 100lvh;
}
```
