# Promise와 setTimeout이 섞인 코드의 출력 순서 정리

## 핵심 요약

- 최종 출력 순서는 **`9 → 5 → 8 → 7 → 1 → 4 → 3 → 6 → 2`** 입니다.
- 이벤트 루프는 **동기 코드 → 마이크로태스크 큐(전부) → 매크로태스크 큐(하나)** 순서로 반복 처리합니다.

## 문제 코드

```js
setTimeout(() => {
  console.log("1");

  setTimeout(() => {
    console.log("2");
  });

  Promise.resolve().then(() => console.log("3"));

  console.log("4");
});

Promise.resolve().then(() => {
  console.log("5");

  setTimeout(() => {
    console.log("6");
  });

  Promise.resolve().then(() => console.log("7"));

  console.log("8");
});

console.log("9");
```

## 정답(출력 결과)

```text
9 → 5 → 8 → 7 → 1 → 4 → 3 → 6 → 2
```

## 이벤트 루프 처리 규칙

이벤트 루프는 아래 과정을 반복하여 태스크를 처리합니다.

1. **동기 코드를 순차적으로 처리**
2. **마이크로태스크 큐의 모든 태스크를 처리** (큐가 비워질 때까지)
3. **매크로태스크 큐에서 하나의 태스크를 가져와 처리**
4. 다시 1번으로 돌아감

## 실행 흐름(단계별)

### 1) 전역(동기) 코드 실행

- 전역 실행 컨텍스트의 동기 코드인 `console.log('9')`가 먼저 실행되어 **`1️⃣ 9`** 출력

### 2) 전역에서 등록된 마이크로태스크 처리

- 전역의 `Promise.resolve().then(...)` 핸들러가 **마이크로태스크 큐**에서 실행
- 핸들러 내부의 동기 코드가 순차 실행되어 **`2️⃣ 5`**, **`3️⃣ 8`** 출력

### 3) (2)에서 추가로 등록된 마이크로태스크 처리

- 같은 `then(...)` 내부에서 등록된 `Promise.then()`이 마이크로태스크 큐에서 실행
- `console.log('7')`이 실행되어 **`4️⃣ 7`** 출력

### 4) 매크로태스크(setTimeout) 처리

- 먼저 등록된 첫 번째 `setTimeout(...)` 콜백이 **매크로태스크 큐**에서 실행
- 콜백 내부 동기 코드가 순차 실행되어 **`5️⃣ 1`**, **`6️⃣ 4`** 출력

### 5) (4)에서 등록된 마이크로태스크 처리

- 동일 `setTimeout(...)` 콜백 내부에서 등록된 `Promise.then()`이 마이크로태스크 큐에서 실행
- `console.log('3')`이 실행되어 **`7️⃣ 3`** 출력

### 6) 남은 매크로태스크 처리(등록 순서대로)

- 남아 있는 매크로태스크 중 먼저 등록된 `console.log('6')`이 실행되어 **`8️⃣ 6`** 출력
- 마지막으로 `console.log('2')`가 실행되어 **`9️⃣ 2`** 출력

## 주의사항 및 실무 팁

- `Promise.then()`은 **마이크로태스크**, `setTimeout()`은 **매크로태스크**로 처리됩니다.
- **한 번 매크로태스크를 처리한 뒤에는**, 그 과정에서 쌓인 **마이크로태스크를 “전부” 비우고** 다음 매크로태스크로 넘어갑니다.
- 디버깅할 때는 “지금 추가된 작업이 마이크로태스크인지/매크로태스크인지”를 먼저 구분하면 순서를 쉽게 예측할 수 있습니다.
