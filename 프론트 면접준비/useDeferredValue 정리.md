# useDeferredValue 정리

## 핵심 요약

- `useDeferredValue`는 값의 반영 우선순위를 낮춰서, 입력처럼 급한 업데이트를 먼저 보여주게 돕는 React 훅이다.
- 무거운 리스트나 검색 결과가 같이 움직일 때 입력이 버벅이는 문제를 줄일 때 특히 유용하다.
- 데이터를 줄여 주는 기능은 아니고, "늦게 보여줘도 되는 UI"를 뒤로 미루는 기능에 가깝다.
- 면접에서는 `useTransition`과 비교해서 "상태 업데이트를 미루는가, 값의 반영을 미루는가" 정도로 구분해 말하면 전달력이 좋다.

## useDeferredValue란?

React에서 어떤 입력값이 바뀔 때 그 값을 바로 쓰는 UI도 있고, 조금 늦게 따라와도 되는 UI도 있다.  
예를 들어 검색창에 타이핑할 때 입력창 자체는 즉시 반응해야 하지만, 아래에 붙은 무거운 검색 결과 리스트는 한 박자 늦어도 사용자는 크게 불편하지 않다.

`useDeferredValue`는 이런 상황에서 "이 값은 조금 늦게 반영해도 된다"는 힌트를 React에 주는 훅이다.  
즉, 최신 값과 완전히 같은 의미이지만 렌더링 우선순위는 더 낮게 다루도록 만든다고 보면 된다.

## 왜 필요한가?

입력값 하나가 바뀔 때마다 큰 리스트가 다시 렌더링되면 사용자는 가장 먼저 입력 지연을 느낀다.  
특히 자동완성, 필터, 대시보드 검색처럼 입력과 결과가 강하게 연결된 화면에서 자주 보인다.

이때 중요한 건 "결과를 완전히 나중에 계산한다"가 아니라 "사용자 입력 반응성을 먼저 확보한다"는 점이다.  
그래서 `useDeferredValue`는 성능 최적화라기보다 사용자 체감 성능을 다루는 도구라고 설명하면 이해가 쉽다.

## 기본 흐름

보통은 입력값은 즉시 상태에 반영하고, 무거운 UI는 지연된 값을 기준으로 그린다.

```jsx
import { useDeferredValue, useState } from "react";

function SearchPage() {
  const [query, setQuery] = useState("");
  const deferredQuery = useDeferredValue(query);

  return (
    <>
      <input
        value={query}
        onChange={(e) => setQuery(e.target.value)}
      />
      <SearchResults query={deferredQuery} />
    </>
  );
}
```

이 구조에서는 사용자가 입력한 최신 문자열은 `query`에 바로 들어가고, `SearchResults`는 `deferredQuery`를 기준으로 조금 늦게 갱신될 수 있다.

## useTransition과 차이

둘 다 우선순위를 다루지만 초점이 다르다.

- `useTransition`은 "이 상태 업데이트는 급하지 않다"라고 표시하는 방식이다.
- `useDeferredValue`는 "이 값은 늦게 따라와도 된다"라고 표시하는 방식이다.

실무에서는 입력값 자체는 즉시 유지하고 싶을 때 `useDeferredValue`가 더 자연스럽다.  
반대로 어떤 상태 변경을 묶어서 비긴급 작업으로 처리하고 싶다면 `startTransition` 쪽이 더 직접적이다.

## 어떤 상황에서 잘 맞나?

- 검색 입력과 결과 리스트가 같이 움직이는 화면
- 대용량 테이블 필터링
- 차트, 그래프처럼 렌더링 비용이 큰 UI
- 텍스트 입력은 부드럽게 유지하고 결과만 늦게 갱신해도 되는 경우

반대로 아주 가벼운 화면에서는 굳이 쓸 이유가 없다.  
지연이라는 개념이 들어가므로, 실제 병목이 있을 때만 적용하는 편이 맞다.

## 면접에서 어떻게 말하면 좋을까?

핵심은 "입력 반응성 보호"라는 표현이다.

`useDeferredValue`는 무거운 UI 때문에 입력이 끊기는 상황에서, 최신 값을 바로 다 쓰지 않고 우선순위를 낮춘 값으로 일부 UI를 렌더링하게 도와주는 훅입니다. 그래서 사용자가 타이핑할 때 입력창은 먼저 부드럽게 반응하고, 결과 리스트는 조금 늦게 따라오게 만들 수 있습니다.

## 면접 답변 예시

> `useDeferredValue`는 React에서 값의 반영 우선순위를 낮춰서, 급하지 않은 UI 업데이트를 뒤로 미루게 도와주는 훅입니다. 대표적으로 검색창처럼 입력은 즉시 반응해야 하지만 아래 결과 리스트는 조금 늦게 바뀌어도 되는 경우에 유용합니다. 이때 입력값은 바로 상태에 저장하고, 무거운 컴포넌트에는 `useDeferredValue`로 만든 값을 넘겨서 사용자 체감 성능을 높일 수 있습니다. `useTransition`이 상태 업데이트 자체를 비긴급 작업으로 표시하는 개념이라면, `useDeferredValue`는 특정 값의 소비를 늦춘다는 점에서 차이가 있습니다.

## 장점

- 입력 지연을 줄여서 사용자 체감 성능을 개선하기 좋다.
- 검색, 필터, 자동완성 같은 인터랙션에서 적용 포인트가 분명하다.
- 상태 구조를 크게 바꾸지 않고도 우선순위를 조절할 수 있다.

## 단점

- 값이 늦게 반영되므로 화면 일부가 잠깐 이전 상태를 보여줄 수 있다.
- 가벼운 화면에서는 효과가 거의 없고 코드 복잡도만 늘 수 있다.
- 데이터 fetching, 캐싱, 가상화 같은 근본 최적화를 대신해 주지는 못한다.

## 주의사항 / 실무 팁

- 먼저 실제 병목이 렌더링인지, 네트워크인지 구분하고 써야 한다.
- 결과 영역이 잠깐 이전 값을 보여줄 수 있으니 로딩 상태나 안내 문구를 같이 설계하면 좋다.
- `useDeferredValue`를 넣기 전에 리스트 가상화, 메모이제이션, 서버 검색 최적화 같은 기본기부터 확인하는 편이 낫다.
