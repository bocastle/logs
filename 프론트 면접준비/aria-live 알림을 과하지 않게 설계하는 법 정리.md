# aria-live 알림을 과하지 않게 설계하는 법 정리

## 핵심 요약

- 비시각 사용자에게 비동기 상태 변화를 전달한다.
- 자주 바뀌는 값을 live region에 넣으면 읽기가 방해된다.
- 상태 메시지 전용 영역을 하나 둔다.

## 개념 설명

`aria-live`는 화면에 새로 생긴 상태 메시지를 스크린 리더가 읽도록 알리는 접근성 속성이다.

`polite`는 현재 읽기를 방해하지 않고, `assertive`는 급한 오류처럼 즉시 알려야 할 때만 쓴다. 같은 메시지를 반복 삽입하면 알림 피로가 생긴다.

## 예시

```html
<p role="status" aria-live="polite" id="save-status">저장되었습니다.</p>
```

저장 완료처럼 흐름을 끊지 않아도 되는 상태는 `polite`가 알맞다. 폼 제출 실패처럼 즉시 수정해야 하는 오류는 필드 연결까지 함께 봐야 한다.

## 면접 답변 예시

> 저장, 로딩, 오류 상태를 화면 구조 안에 남길 수 있다. DOM에서 제거되는 시간이 빠르면 읽히지 않을 수 있다. 스크린 리더에서 실제 읽기 순서를 확인한다.

## 장점

- 토스트 메시지의 의미를 보조 기술에도 맞출 수 있다.

## 단점

- `assertive` 남용은 더 중요한 알림을 가린다.

## 주의사항 / 실무 팁

- 진행률처럼 빈번한 값은 변화 폭을 줄여 알린다.
