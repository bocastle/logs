# content-visibility 정리

## 핵심 요약
- `content-visibility`는 화면에 아직 보이지 않는 영역의 렌더링 비용을 브라우저가 줄이도록 돕는 CSS 속성이다.
- 긴 페이지나 카드 목록처럼 DOM은 많지만 실제로 당장 보이는 영역은 제한적인 화면에서 특히 효과적이다.
- 면접에서는 "가상화와 비슷한 목표를 CSS 레벨에서 일부 지원하는 최적화"라고 설명하면 이해가 쉽다.
- 다만 접근성, 레이아웃 안정성, 측정 방식까지 함께 고려해야 해서 무조건 켜두는 옵션은 아니다.

## 개념 설명
화면 아래쪽에 있는 콘텐츠는 사용자가 아직 보지 않았는데도, 브라우저는 기본적으로 레이아웃 계산과 페인트 준비를 해두려 한다. 콘텐츠가 많아질수록 초기 렌더링 비용이 커지는 이유다.

`content-visibility`는 "지금 안 보이는 영역은 일단 덜 계산해도 된다"는 힌트를 브라우저에 주는 속성이다. 특히 `auto`를 쓰면 뷰포트 바깥의 요소에 대해 렌더링 비용을 줄일 수 있어서 긴 문서형 페이지나 피드 화면 최적화에 자주 언급된다.

다만 이것은 DOM을 없애는 기술이 아니라, 렌더링 비용을 늦추는 기술에 가깝다. 그래서 면접에서는 리스트 가상화처럼 아예 노드를 줄이는 전략과 구분해서 설명하는 것이 중요하다.

## 예시
공지사항 페이지에 긴 카드 목록이 있고, 한 번에 50개 섹션이 렌더링된다고 가정해보자.

- 최적화 전
  - 첫 진입 시 아래쪽 카드까지 전부 레이아웃과 페인트 비용이 발생할 수 있다
  - 초기 로딩이 무거워질 수 있다
- `content-visibility` 적용 후
  - 현재 보이는 영역 중심으로 렌더링이 진행된다
  - 스크롤 전까지는 아래쪽 콘텐츠 비용을 줄일 수 있다

이런 식으로 초기 표시 성능을 개선할 수 있지만, 스크롤 시점에 콘텐츠가 자연스럽게 나타나는지도 함께 확인해야 한다.

## 면접 답변 예시
- 질문: "`content-visibility`는 무엇인가요?"  
  답변: "`content-visibility`는 화면 밖에 있는 요소의 렌더링 비용을 줄이도록 브라우저에 힌트를 주는 CSS 속성입니다. 특히 긴 페이지에서 초기 렌더링 부담을 줄이는 데 도움이 됩니다."
- 질문: "리스트 가상화와 같은 건가요?"  
  답변: "목표는 비슷하지만 방식이 다릅니다. 리스트 가상화는 DOM 자체를 줄이는 전략이고, `content-visibility`는 DOM은 유지한 채 브라우저가 지금 당장 필요 없는 렌더링 계산을 덜 하게 만드는 CSS 최적화입니다."

## 장점
- 초기 렌더링 비용을 줄여 긴 페이지의 체감 성능 개선에 도움이 된다
- 자바스크립트 없이 CSS만으로 적용 가능한 최적화라는 점이 간단하다
- 문서형 페이지, 피드, 카드 목록처럼 아래쪽 콘텐츠가 많은 화면에서 효과를 보기 쉽다

## 단점
- 모든 UI에 만능처럼 적용되지는 않는다
- 레이아웃 안정성을 위해 `contain-intrinsic-size` 같은 보조 설정을 함께 고려해야 한다
- 접근성 도구나 측정 도구 관점에서 의도와 다르게 보일 수 있어 검증이 필요하다

## 주의사항 / 실무 팁
- `content-visibility: auto`만 넣고 끝내지 말고, 레이아웃 점프를 줄이기 위한 크기 힌트도 함께 검토하는 편이 좋다.
- 실제 효과는 긴 페이지나 반복 카드 UI에서 더 잘 나타난다. 작은 화면에서는 체감 차이가 크지 않을 수 있다.
- 면접에서는 리스트 가상화, lazy loading, CSS containment와 차이를 비교해서 말하면 이해도가 높아 보인다.
- 실무에서는 Lighthouse 점수보다 실제 스크롤 경험과 CLS 같은 지표를 함께 확인해야 한다.
