# Popover API 정리

## 핵심 요약
- `Popover API`는 브라우저가 기본적으로 팝오버 UI의 열기와 닫기 동작을 지원하도록 만든 웹 플랫폼 기능이다.
- 툴팁, 메뉴, 도움말 패널, 간단한 오버레이 UI를 만들 때 JS 상태 관리 부담을 줄일 수 있다.
- 면접에서는 "모달보다 가볍고, 커스텀 드롭다운보다 브라우저 기본 동작을 더 활용하는 API"라고 설명하면 좋다.
- 장점만 말하지 말고 브라우저 지원 범위, 접근성 설계, 복잡한 포지셔닝 한계도 함께 짚어야 답변이 균형 잡힌다.

## 개념 설명
프론트엔드에서 팝오버 UI는 자주 등장한다. 예를 들어 사용자 메뉴, 필터 패널, 도움말 말풍선, 간단한 액션 메뉴 같은 것들이다. 보통 이런 UI는 열림 상태 관리, 바깥 클릭 닫기, Escape 닫기, 포커스 이동 같은 로직을 직접 구현해야 했다.

`Popover API`는 이런 반복 작업 일부를 브라우저가 처리하게 해 준다. 요소에 `popover` 속성을 붙이고, 버튼에서 `popovertarget`을 연결하면 비교적 적은 코드로 열고 닫는 동작을 만들 수 있다.

핵심은 "팝오버라는 UI 의도를 HTML 차원에서 선언한다"는 점이다. 그래서 단순히 `display: none`을 토글하는 것보다 의미가 더 분명하고, 브라우저 기본 동작을 활용할 수 있다.

다만 이 API가 모든 오버레이 문제를 해결하는 것은 아니다. 복잡한 포커스 트랩, 애니메이션, 정교한 위치 계산이 필요한 경우에는 여전히 추가 설계가 필요하다.

## 예시
예를 들어 프로필 아바타를 누르면 작은 사용자 메뉴가 열리는 UI를 생각해 보자.

- 기존 방식은 `isOpen` 상태, 문서 클릭 감지, Escape 처리, aria 속성 동기화를 직접 구현한다
- `Popover API`를 쓰면 버튼과 대상 요소를 선언적으로 연결해 기본 열기/닫기 동작을 빠르게 만들 수 있다
- 여기에 필요한 스타일만 얹으면 가벼운 메뉴 UI를 비교적 단순하게 구성할 수 있다

특히 디자인 시스템 초기에 "일단 안전하게 동작하는 기본 팝오버"를 만들고 싶을 때 생산성이 좋다.

## 면접 답변 예시
- 질문: "`Popover API`는 무엇인가요?"  
  답변: "브라우저가 팝오버 UI의 기본 열기와 닫기 동작을 지원하도록 만든 웹 플랫폼 기능입니다. 버튼과 팝오버 요소를 선언적으로 연결해서 메뉴나 도움말 패널 같은 UI를 비교적 적은 코드로 만들 수 있습니다."
- 질문: "언제 유용한가요?"  
  답변: "가벼운 드롭다운, 도움말, 액션 메뉴처럼 오버레이가 필요하지만 모달처럼 무거운 제어까지는 필요하지 않을 때 유용합니다."
- 질문: "주의할 점은 무엇인가요?"  
  답변: "브라우저 지원 범위를 확인해야 하고, 접근성 요구사항이나 복잡한 위치 계산이 있으면 별도 보완이 필요합니다."

## 장점
- 팝오버 의도를 HTML에 직접 표현할 수 있어 코드가 단순해진다
- 열기, 닫기, 바깥 클릭 처리 같은 기본 동작을 브라우저에 맡길 수 있다
- 가벼운 오버레이 UI를 빠르게 만들 때 생산성이 좋다

## 단점
- 복잡한 포지셔닝과 정교한 상호작용은 여전히 추가 구현이 필요하다
- 브라우저 지원 범위를 확인하지 않으면 예상과 다른 동작이 나올 수 있다
- 기존 헤드리스 UI 라이브러리만큼 세밀한 제어를 바로 제공하지는 않는다

## 주의사항 / 실무 팁
- `Popover API`를 쓴다고 접근성 문제가 자동 해결되는 것은 아니다. 역할, 라벨, 키보드 이동 흐름은 직접 점검해야 한다.
- 모달, 드롭다운, 툴팁을 모두 같은 방식으로 보면 안 된다. 어떤 상호작용 수준이 필요한지 먼저 구분해야 한다.
- 면접에서는 "`직접 JS로 상태를 다 관리하던 팝오버를 브라우저 기본 기능으로 일부 대체하는 API`"라고 설명하면 이해가 빠르다.
- 실무에서는 `Anchor Positioning`, 디자인 시스템 컴포넌트, 폴백 전략을 함께 고려해야 안정적으로 도입할 수 있다.
