# 자바스크립트 생성자 함수와 `class` 문법 정리

## 핵심 요약

- **생성자 함수**는 `function`으로 정의한 함수를 `new`와 함께 호출해 **객체를 생성**하는 방식입니다.
- 생성자 함수의 메서드는 보통 `prototype`에 붙여 **공유**합니다.
- `class` 문법은 생성자 함수 기반 프로토타입 패턴의 **가독성/유지보수성 문제를 완화**하기 위해 도입되었고, `extends`, `super`, `static`, `getter/setter` 등을 더 직관적으로 제공합니다.
- `class`는 생성자 함수와 달리 **일반 함수처럼 호출이 불가**(반드시 `new` 필요)하도록 제약이 있습니다.

## 생성자 함수(Constructor Function)란?

### 개념

자바스크립트에서 생성자 함수는 **객체를 생성하는 하나의 방법**입니다. 일반적으로 `function` 키워드로 정의하고, `new` 키워드로 호출하면 새로운 객체가 만들어집니다.

- 생성자 함수 내부의 `this`는 **새롭게 생성된 객체**를 가리킵니다.
- `this`에 속성을 추가하면 해당 객체에 저장됩니다.

### 예시 코드

```js
function Person(name, age) {
  this.name = name;
  this.age = age;
}

Person.prototype.greet = function () {
  console.log(`안녕하세요, 저는 ${this.name}입니다.`);
};

const person1 = new Person("Alice", 25);
```

## `class` 문법은 왜 도입되었나요?

### 배경

생성자 함수 방식은 유지보수성이 떨어질 수 있습니다.

- 명확한 클래스 개념이 없어 상속 구현 시 **프로토타입 체인**을 직접 다뤄야 하며, 가독성이 좋지 않을 수 있습니다.
- 다른 객체지향 언어와 형태가 많이 달라 이해하기 비교적 어렵습니다.
- `new` 키워드 없이 일반 함수처럼 호출될 수도 있어 혼동을 유발할 수 있습니다.

이러한 단점을 극복하기 위해 `class` 문법이 등장했습니다.

### 예시 코드

```js
class Person {
  constructor(name, age) {
    this.name = name;
    this.age = age;
  }

  greet() {
    console.log(`안녕하세요, 저는 ${this.name}입니다.`);
  }
}

const person2 = new Person("Bob", 30);
```

### `class`가 제공하는 개선점

- 생성자와 메서드를 더 **명확하고 직관적으로** 정의할 수 있습니다.
- 다른 객체지향 언어와 유사해 **이해가 쉬운 형태**입니다.
- `extends`, `super`를 이용해 **상속을 간결하게** 구현할 수 있습니다.
- `static`, `getter/setter` 등 객체지향 관련 키워드를 지원합니다.
- 생성자 함수와 달리 일반 함수처럼 호출할 수 없도록 하는 **제약(반드시 `new`)** 이 추가됩니다.

## 장점

### 생성자 함수의 장점

- 오래전부터 사용된 방식으로, 프로토타입 기반 메서드 공유(`prototype`)를 통해 **메모리 효율**을 얻을 수 있습니다.

### `class`의 장점

- 선언/상속/메서드 정의가 **가독성 좋게** 정리되어 유지보수에 유리합니다.
- `extends`, `super`, `static`, `getter/setter` 등으로 **객체지향 표현이 간결**해집니다.
- **`new` 없이 호출 불가**한 제약으로 실수를 줄일 수 있습니다.

## 단점

### 생성자 함수의 단점

- 상속/확장 시 프로토타입 체인을 직접 다루는 코드가 들어가면 **가독성이 떨어질 수** 있습니다.
- `new` 누락 등 호출 방식 혼동이 발생할 수 있습니다.

### `class`의 단점

- 원문에서는 별도의 단점이 직접적으로 언급되지는 않았습니다.

## 주의사항 / 실무 팁

- “생성자 함수 vs `class`”는 문법이 다를 뿐, 자바스크립트는 기본적으로 **프로토타입 기반**임을 같이 이해하면 좋습니다.
- 팀/프로젝트에서는 보통 `class` 문법을 사용하면 **표현이 통일**되어 협업에 유리합니다.
- 생성자 함수를 쓸 때는 실수 방지를 위해 `new` 누락 가능성을 항상 염두에 두고, 패턴/린트 규칙으로 방어하는 것이 좋습니다.
