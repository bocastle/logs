# JSON Schema 정리

## 핵심 요약

- JSON Schema는 **JSON 데이터의 형식을 기술하고 검증**하기 위한 명세서입니다.
- 특정 JSON이 어떤 **구조/필드/타입/필수 여부**를 가져야 하는지 정의할 수 있습니다.
- API 데이터 검증, TypeScript 타입 자동 생성, 에디터 설정 파일 자동 완성 등으로 활용됩니다.

## JSON Schema란?

JSON Schema는 JSON 데이터의 형식을 기술하고 검증하기 위한 명세서입니다.  
특정 JSON이 어떤 구조를 가져야 하는지를 명시할 수 있도록 해줍니다.

## 예시: 회원 정보 스키마

아래처럼 회원 정보에 대한 명세를 작성할 수 있습니다.

```json
{
  "type": "object",
  "properties": {
    "username": { "type": "string", "minLength": 3 },
    "email": { "type": "string", "format": "email" },
    "password": { "type": "string", "minLength": 6 }
  },
  "required": ["username", "email", "password"],
  "additionalProperties": false
}
```

## JSON Schema 활용 사례

### 백엔드 API 통신에서 데이터 형식 검증

백엔드에서 데이터를 내려줄 때 JSON Schema로 다음을 정의할 수 있습니다.

- 어떤 속성들을 가지는지
- 타입은 무엇인지
- 필수인지 아닌지

프론트엔드는 그 스키마를 기준으로 데이터 유효성을 검증할 수 있어, API 통신에서 발생할 수 있는 오류를 사전에 방지할 수 있습니다.

### 정적 타입 생성 도구와 통합

JSON Schema는 정적 타입 생성 도구와 통합되어 개발 생산성을 높여주기도 합니다.

예를 들어 `json-schema-to-typescript` 같은 도구를 사용하면 JSON Schema로부터 TypeScript 타입을 자동 생성할 수 있습니다.  
이는 API 명세에 따라 타입을 작성/수정하는 시간을 아껴주며, 스키마와 타입 간의 불일치를 줄이는 데 도움이 됩니다.

### 설정 파일 명세로 활용 (에디터 자동 완성)

`eslintrc`, `tsconfig`, `prettierrc` 같은 설정 파일들은 대부분 JSON 기반 형식을 사용합니다.  
`JSON Schema Store`에서 이러한 설정 파일들의 JSON Schema를 찾아볼 수 있습니다.

VSCode 같은 에디터에서 이 스키마를 기반으로 자동 완성, 타입 힌트, 경고 메시지 등을 지원하므로 설정 실수를 줄이고 생산성을 높이는 데 도움이 됩니다.

## 장점

- 데이터 구조/제약 조건을 명확히 정의해 **유효성 검증**에 활용할 수 있습니다.
- 스키마를 기준으로 타입을 자동 생성해 **개발 생산성**을 높일 수 있습니다.
- 에디터 자동 완성/힌트/경고를 통해 **설정 실수**를 줄일 수 있습니다.

## 단점

- 원문에서는 별도의 단점이 직접적으로 언급되지는 않았습니다.

## 주의사항 / 실무 팁

- API 응답/요청에 스키마를 적용하면, 프론트엔드에서 데이터 유효성 검증 흐름을 표준화하기 좋습니다.
- JSON Schema로부터 타입을 생성하는 경우, 스키마가 변경될 때 타입도 함께 갱신되도록(예: 스크립트/CI) 관리하면 불일치를 줄일 수 있습니다.
- 설정 파일 자동 완성은 스키마 연결이 핵심이므로, 프로젝트에서 사용하는 설정 파일의 스키마를 `JSON Schema Store`에서 찾아 적용하는 방식을 고려해 보세요.
