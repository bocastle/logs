# `<a>` 태그로 외부 페이지를 열 때 보안 고려사항 정리

## 핵심 요약

- `target="_blank"`로 외부 페이지를 열 때는 **탭 내빙(Tabnabbing)** 및 **Referer(리퍼러) 정보 유출**을 막기 위해 `rel` 속성을 적절히 설정해야 합니다.
- 보통 `rel="noopener noreferrer"`를 함께 적용하며, 상황에 따라 `nofollow`도 추가합니다.

## 개요

`<a>`를 이용해 외부 링크를 연결할 때는 `rel` 속성을 적절히 설정하여 중요 정보가 유출되지 않도록 해야 합니다.

## 보안/운영 측면에서 중요한 `rel` 속성들

### `rel="noopener"`

- 외부 페이지에서 `opener` 객체에 접근하지 못하도록 막습니다.
- 기본적으로 `target="_blank"`가 설정된 `<a>`를 통해 열린 외부 사이트는 `window.opener` 객체에 접근할 수 있습니다.
- `window.opener` 접근을 허용하면, 악의적으로 피싱 사이트로 연결시키는 **탭 내빙(Tabnabbing) 공격**이 가능해집니다.
- 이를 방지하기 위해 `noopener`를 설정해야 합니다.

### `rel="noreferrer"`

- 링크 연결 시 **Referer 헤더 정보 노출**을 막습니다.
- 브라우저는 링크 연결 시 Referer 헤더를 통해 사용자가 어떤 웹사이트에서 이동된 것인지를 전달합니다.
- 이 정보에 민감한 데이터가 포함될 수 있습니다.
  - 예: URL에 세션 ID나 사용자 식별 정보가 포함되어 있다면 외부로 유출될 수 있음
- 따라서 `noreferrer`를 설정하여 이를 예방해야 합니다.
- 참고로 `noreferrer`에는 `noopener`와 마찬가지로 `opener`를 생략하는 기능도 포함됩니다.

### `rel="nofollow"`

- 검색 엔진이 외부 링크를 따라가지 않도록 설정합니다.
- 기본적으로 검색 엔진은 페이지 내 링크를 따라가며 웹을 크롤링합니다.
- 검증되지 않은 외부 링크나 상업적 링크가 검색 엔진에 의해 인덱싱되면 SEO 상 불리할 수 있습니다.
- 예:
  - 사용자 생성 컨텐츠(UGC)에서 스팸 링크 방지
  - 광고 링크가 SEO에 영향을 미치지 않도록 처리

## 설정 예시

세 가지 속성은 함께 적용할 수 있습니다.

```html
<a
  href="https://example.com"
  target="_blank"
  rel="noopener noreferrer nofollow"
>
  외부 링크 열기
</a>
```

## 탭 내빙(Tabnabbing) 공격이란?

- 사용자가 새롭게 열린 탭에서 기존 탭으로 돌아올 때, 해당 탭을 원본 사이트와 유사한 피싱 사이트로 이동시켜 공격을 시도하는 기법입니다.
- 사용자가 외부 링크를 클릭하여 새로운 탭이 열리고, 이후 원래 탭으로 돌아왔을 때 해당 페이지가 악성 사이트로 변경될 수 있습니다.
- 이를 방지하기 위해 `rel="noopener"`를 설정하면 새로운 탭이 원본 탭의 컨트롤을 가질 수 없게 되어 공격이 불가능해집니다.

## 장점

- `noopener` 적용 시 탭 내빙 공격 위험을 낮출 수 있습니다.
- `noreferrer` 적용 시 Referer 기반 정보 유출을 예방할 수 있습니다.
- `nofollow` 적용 시 검증되지 않은 외부 링크로 인한 SEO 리스크를 줄일 수 있습니다.

## 단점

- `noreferrer`를 사용하면 Referer 정보가 전달되지 않아, 외부 사이트에서 유입 분석(레퍼럴 기반 분석)이 제한될 수 있습니다.
- `nofollow`는 검색 엔진이 링크를 따라가지 않도록 하므로, 의도한 SEO 신호 전달이 필요할 때는 부적합할 수 있습니다.

## 주의사항 / 실무 팁

- 외부 링크를 `target="_blank"`로 열어야 한다면 기본값으로 `rel="noopener noreferrer"`를 우선 고려합니다.
- 사용자 입력(UGC)로 생성되는 외부 링크에는 스팸/광고 링크가 섞일 수 있으므로 `nofollow`를 함께 적용하는 전략이 유용합니다.
- URL에 세션 ID 등 민감 정보가 포함되지 않도록 설계(쿼리스트링에 민감 정보 금지)하는 것이 근본적인 예방책입니다.
