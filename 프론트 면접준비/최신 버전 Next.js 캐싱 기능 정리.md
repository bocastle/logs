# 최신 버전 Next.js 캐싱 기능 정리

## 핵심 요약

- 최신 Next.js는 성능 최적화를 위해 캐싱을 **리퀘스트 메모이제이션**, **클라이언트 라우터 캐시**, **데이터 캐시**, **풀 라우트 캐시**로 나눠 제공합니다.
- `fetch()`에 캐시 옵션(`cache`, `revalidate`)을 적용해 **응답 캐싱/갱신 주기/미캐싱**을 제어할 수 있습니다.
- Dynamic Function(`cookies()`, `headers()`) 사용 여부와 설정에 따라 **풀 라우트 캐시 활성화 여부**가 달라집니다.

## 개요: Next.js가 제공하는 캐싱 기능들

최신 버전의 Next.js는 성능 최적화를 위해 다양한 캐싱 기능들을 제공합니다. 크게 아래 4가지로 나눠 설명할 수 있습니다.

1. 리퀘스트 메모이제이션 (Request Memoization)
2. 클라이언트 라우터 캐시 (Client Router Cache)
3. 데이터 캐시 (Data Cache)
4. 풀 라우트 캐시 (Full Route Cache)

## 1) 리퀘스트 메모이제이션

동일한 요청이 여러 번 발생할 때, **불필요한 추가 요청을 방지**하는 기능입니다.

- 기본적으로 Next.js는 **서버 사이드에서 동일한 요청이 중복 발생**하면 기존 응답을 재사용합니다.
- 하나의 페이지 내에 존재하는 **중복 요청을 방지**하여 네트워크 비용을 줄입니다.

## 2) 클라이언트 라우터 캐시

레이아웃, 로딩 상태 등 공통적으로 쓰이는 요소에 대한 **RSC Payload를 브라우저에 캐싱**하는 기능입니다.

- 클라이언트 측 페이지 전환 시 **캐싱된 데이터를 활용**해 빠른 탐색이 가능합니다.
- **서버 부하 감소** 효과가 있습니다.

## 3) 데이터 캐시

네트워크 요청에 대한 응답을 **Next 서버에 캐싱**하는 기능입니다.  
Next.js에서는 `fetch()`에 다양한 데이터 캐시 옵션을 적용할 수 있습니다.

### `fetch()` 캐시 옵션 예시

```ts
// 캐싱 적용
fetch("https://example.com/api", { cache: "force-cache" });

// 캐싱 적용 + 일정 주기로 갱신 (revalidate)
fetch("https://example.com/api", { next: { revalidate: 60 } });

// 캐싱 없이 항상 최신 데이터
fetch("https://example.com/api", { cache: "no-store" });
```

## 4) 풀 라우트 캐시

서버에서 미리 생성된 페이지를 저장해두고, 이후 동일한 요청이 들어오면 **캐싱된 페이지를 즉시 제공**하는 방식입니다.

- `cookies()`, `headers()` 같은 **Dynamic Function**을 사용하지 않거나,
- Route Segment Config를 동적 옵션으로 설정하지 않고,
- 데이터 캐시를 적용하면 **풀 라우트 캐시가 활성화**됩니다.
- `fetch()`에 `revalidate` 옵션을 설정하면 일정 주기로 페이지가 갱신되는 **ISR 방식**으로 동작하도록 설정할 수도 있습니다.
- 적용 시 **페이지 로드 속도를 극대화**할 수 있습니다.

## 장점

### 캐싱 기능 전반의 장점

- 중복 요청을 줄여 **네트워크 비용 감소**에 도움이 됩니다.
- 페이지 전환/로딩이 빨라져 **사용자 경험 개선**에 유리합니다.
- 서버가 동일 연산/응답을 반복하지 않아 **서버 부하를 줄일 수 있습니다.**

## 단점

### 캐싱 적용 시 주의되는 점

- 캐시가 과도하게 적용되면 **데이터가 최신 상태가 아닐 수 있습니다.**
- 페이지/데이터의 특성에 따라 `revalidate` 주기나 `no-store` 적용 등 **설계가 필요**합니다.

## 주의사항 / 실무 팁

- 사용자/요청별로 달라지는 정보(예: 로그인 상태, 개인화 데이터)는 `cookies()`, `headers()` 같은 Dynamic Function과 연관될 수 있으므로 **풀 라우트 캐시와의 관계를 반드시 점검**하세요.
- 데이터가 반드시 최신이어야 하면 `cache: "no-store"`를 고려하고, 갱신 주기를 두어도 괜찮으면 `revalidate`로 균형을 잡으세요.
- 같은 페이지 내에서 동일 API를 여러 번 호출하는 구조라면 리퀘스트 메모이제이션이 효과를 볼 수 있으니 **중복 호출 패턴을 먼저 점검**하세요.
