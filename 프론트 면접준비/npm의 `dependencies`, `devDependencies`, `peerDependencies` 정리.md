# npm의 `dependencies`, `devDependencies`, `peerDependencies` 정리

## 핵심 요약

- `dependencies`: **프로덕션(운영)에서도 필요한 패키지**
- `devDependencies`: **개발 과정에서만 필요한 도구/패키지**
- `peerDependencies`: 라이브러리 등이 **호환을 위해 “함께 설치되어야 하는” 다른 패키지의 버전 범위**를 명시

## 개요

`dependency`, `devDependency`, `peerDependency`는 모두 npm에서 사용되는 의존성 유형입니다.

## 의존성 유형별 설명

### `dependencies`

프로덕션 환경에서도 필요한 패키지를 의미합니다.

- 예: `react`를 사용하는 프로젝트라면 `react`는 실행에 필수이므로 `dependencies`에 포함
- 배포 환경에서도 설치되며, 애플리케이션 동작에 직접적인 영향을 줍니다.

### `devDependencies`

개발 과정에서만 필요한 패키지를 의미합니다.

- 예: `eslint`, `jest`, `webpack`
- 코드 품질 검증, 테스트, 번들링 등에 쓰이지만 실제 서비스 운영에는 필요하지 않습니다.
- 보통 프로덕션 환경에는 설치되지 않도록 `devDependencies`에 포함합니다.

### `peerDependencies`

호환성을 위해 특정 버전의 다른 패키지를 필요로 한다는 사실을 명시합니다.

- 주로 **라이브러리 개발** 시 중요합니다.
- 특정 버전의 다른 패키지를 사용해야 하는 경우 지정합니다.
- 호환성 문제를 줄이면서도 동일 패키지가 여러 번 설치되는 문제를 방지하는 데 도움이 됩니다.

## `dependencies`와 `devDependencies`를 잘못 설정하면 생기는 문제

### 1) `devDependencies`로 넣어야 할 것을 `dependencies`로 넣은 경우

- 불필요한 패키지가 프로덕션 환경에 포함됩니다.
- 예: `webpack` 같은 패키지를 `dependencies`에 넣으면
  - 배포된 애플리케이션 크기가 불필요하게 커질 수 있고
  - 배포 속도 저하 / 서버 부하 증가로 이어질 수 있습니다.
- 또한 불필요한 패키지를 포함하면 보안 취약점이 늘어날 수 있습니다.

### 2) `dependencies`로 넣어야 할 것을 `devDependencies`로 넣은 경우

- 운영 환경에서 해당 패키지가 누락되어 애플리케이션이 정상적으로 실행되지 않을 수 있습니다.

## 장점

### 올바르게 구분했을 때의 장점

- 운영 환경에 불필요한 패키지가 포함되지 않아 **배포 크기/속도/부하** 측면에서 유리합니다.
- 불필요한 의존성이 줄어 **보안 리스크**를 낮출 수 있습니다.
- 개발용 도구와 런타임 의존성이 분리되어 **유지보수**가 쉬워집니다.

## 단점

### 잘못 구분했을 때의 단점

- 운영 환경에서 불필요한 패키지가 설치되거나, 반대로 필요한 패키지가 누락되어 **장애**로 이어질 수 있습니다.
- 불필요한 패키지 포함으로 인해 **보안 취약점** 노출 가능성이 커질 수 있습니다.

## 주의사항 / 실무 팁

- “서비스 실행에 필요하면 `dependencies`”, “개발 도구면 `devDependencies`”로 먼저 구분하세요.
- 라이브러리를 만들 때는 사용자가 이미 설치했을 가능성이 높은 패키지(예: `react`)에 대해 `peerDependencies`로 버전 호환을 명시하는 경우가 많습니다.
- 배포 환경에서 의존성 설치 전략이 다를 수 있으니(예: CI/CD에서 devDependencies 제외) **운영 배포 파이프라인 기준**으로 확인하세요.

## 설정 예시 (`package.json`)

```json
{
  "dependencies": {
    "react": "^18.0.0"
  },
  "devDependencies": {
    "eslint": "^9.0.0",
    "jest": "^29.0.0",
    "webpack": "^5.0.0"
  },
  "peerDependencies": {
    "react": ">=18"
  }
}
```
