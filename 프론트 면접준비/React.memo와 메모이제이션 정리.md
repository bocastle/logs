# React.memo와 메모이제이션 정리

## 핵심 요약

- `React.memo`는 부모가 다시 렌더링되어도 props가 같으면 자식 컴포넌트의 재렌더링을 건너뛰게 도와주는 기능이다.
- 메모이제이션은 "이전 계산 결과를 재사용해서 불필요한 작업을 줄이는 것"이고, React에서는 `React.memo`, `useMemo`, `useCallback`이 대표적이다.
- 면접에서는 "무조건 성능 최적화 도구"라고 말하기보다, `재렌더링 비용이 실제로 큰 곳에서 선택적으로 쓰는 도구`라고 설명하는 편이 좋다.
- 잘못 쓰면 비교 비용과 코드 복잡도만 늘고 체감 이득은 거의 없을 수 있다.

## 메모이제이션이 뭐냐

메모이제이션은 같은 입력이 들어오면 이전 결과를 다시 쓰는 방식이다.  
프론트엔드에서는 비싼 계산을 반복하지 않거나, 불필요한 렌더링을 줄일 때 자주 언급된다.

중요한 건 "다 캐시한다"가 아니라, `반복 비용이 실제로 아까운 지점만 줄이는 전략`이라는 점이다.

## React.memo는 무엇을 줄여주나

React에서는 부모 컴포넌트가 렌더링되면 자식도 기본적으로 다시 렌더링될 수 있다.  
그런데 자식이 받는 props가 바뀌지 않았고, 자식 렌더링 비용이 크다면 이게 낭비가 될 수 있다.

`React.memo`는 함수형 컴포넌트를 감싸서 props가 이전과 같으면 렌더링 결과를 재사용하게 도와준다.  
즉 "부모가 렌더링됨"과 "자식이 꼭 다시 그려져야 함"을 분리하는 장치에 가깝다.

## useMemo, useCallback과는 뭐가 다르냐

### React.memo

컴포넌트 자체의 재렌더링을 줄이는 데 초점이 있다.

### useMemo

비싼 계산 결과를 메모이제이션한다.  
예를 들어 큰 배열 정렬이나 필터링 결과를 다시 계산하지 않도록 할 때 쓸 수 있다.

### useCallback

함수 자체를 메모이제이션한다.  
주로 자식에게 내려주는 콜백이 매번 새로 만들어져서 `React.memo`가 무력화되는 상황을 줄일 때 같이 언급된다.

면접에서는 "컴포넌트, 값, 함수"로 나눠 기억하면 설명이 쉽다.

## 언제 효과가 있나

- 리스트 아이템처럼 같은 모양의 자식이 많을 때
- 차트, 에디터, 복잡한 카드처럼 자식 렌더링 비용이 큰 경우
- 부모 상태는 자주 바뀌지만 자식 props는 거의 안 바뀌는 경우

반대로 자식이 아주 단순하거나 props가 매번 새 객체로 바뀌면 효과가 작다.

## 왜 남발하면 안 되나

메모이제이션도 공짜는 아니다.  
props 비교 비용이 들고, 코드 읽기가 어려워질 수 있다.

예를 들어 작은 버튼 컴포넌트까지 전부 `React.memo`로 감싸면 관리 포인트만 늘고 성능 차이는 거의 없을 수 있다.  
그래서 면접에서는 "프로파일링 없이 습관적으로 넣지는 않는다"라고 말하면 현실감이 있다.

## 실무에서 자주 나오는 함정

`React.memo`를 붙였는데도 리렌더링이 계속되는 경우가 있다.  
대부분은 props로 넘기는 객체나 함수가 매 렌더마다 새로 만들어지기 때문이다.

예를 들어 `{ userId, role }` 같은 객체를 인라인으로 만들거나, 클릭 핸들러를 매번 새 함수로 만들면 얕은 비교에서 달라진 것으로 본다.  
이때는 구조를 바꾸거나 `useMemo`, `useCallback`을 함께 검토해야 한다.

## 면접 답변 예시

> React.memo는 함수형 컴포넌트에서 props가 바뀌지 않았을 때 불필요한 재렌더링을 줄이기 위한 도구입니다. 메모이제이션이라는 건 같은 입력에 대해 이전 결과를 재사용하는 전략이고, React에서는 React.memo, useMemo, useCallback이 대표적입니다. 다만 무조건 붙인다고 좋은 건 아니고, 자식 렌더링 비용이 크거나 부모는 자주 바뀌지만 자식 props는 잘 안 바뀌는 경우에 효과가 큽니다. 반대로 비교 비용과 코드 복잡도가 늘 수 있어서 실제 병목이 있는 지점에 선택적으로 적용하는 게 맞습니다.

## 장점

- React 성능 최적화를 단순 암기 말고 원리 중심으로 설명하기 좋다.
- `React.memo`, `useMemo`, `useCallback`의 차이를 함께 정리할 수 있다.
- "최적화도 비용이 있다"는 균형 잡힌 답변을 만들기 좋다.

## 단점

- 얕은 비교 특성을 모르면 왜 안 먹히는지 설명이 막힐 수 있다.
- 무조건 쓰는 최적화처럼 말하면 실무 감각이 약해 보일 수 있다.
- 성능 병목 확인 없이 적용한다고 답하면 설득력이 떨어진다.

## 주의사항 / 실무 팁

- 면접에서는 `React.memo는 컴포넌트`, `useMemo는 값`, `useCallback은 함수`로 정리하면 전달이 빠르다.
- "부모가 렌더링되어도 자식이 꼭 다시 렌더링될 필요는 없다"는 문장을 기억해 두면 설명이 자연스럽다.
- 객체와 함수 props 때문에 메모이제이션이 쉽게 깨진다는 점을 같이 말하면 좋다.
- 가능하면 React DevTools Profiler 같은 도구로 병목을 확인한 뒤 적용한다고 덧붙이면 실무 답변에 가깝다.
