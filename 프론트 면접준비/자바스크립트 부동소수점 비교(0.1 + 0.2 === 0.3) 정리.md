# 자바스크립트 부동소수점 비교(0.1 + 0.2 === 0.3) 정리

## 핵심 요약

자바스크립트에서 `0.1 + 0.2 === 0.3`의 결과는 **`false`**입니다. 자바스크립트의 숫자는 **IEEE 754 64비트 부동소수점**으로 저장되며, `0.1`, `0.2` 같은 값이 이진수로 **정확히 표현되지 못해 근사값**으로 저장되기 때문에 오차가 발생합니다.

## 실행 결과가 `false`인 이유

- 자바스크립트에서 모든 숫자는 **IEEE 754 표준을 따르는 64비트 부동소수점 형식**으로 저장됩니다.
- `0.1`, `0.2`를 이진수로 변환하면 **무한 반복되는 소수**로 표현되므로, 컴퓨터는 이를 **근사값으로 저장**합니다.
- 이 때문에 실제로는 다음과 같은 값이 되어 비교가 실패합니다.

```js
0.1 + 0.2; // 0.30000000000000004
0.1 + 0.2 === 0.3; // false
```

## 오차를 해결하는 방법

### 반올림/자릿수 제한

- 특정 자릿수까지 반올림하는 방식입니다.
- `toFixed()` 또는 `toPrecision()`을 사용할 수 있습니다.
- 단, **숫자가 문자열로 변환되는 부작용**이 있습니다.

```js
(0.1 + 0.2)
  .toFixed(1)(
    // "0.3"
    0.1 + 0.2,
  )
  .toPrecision(1); // "0.3"
```

### `Number.EPSILON`을 이용한 비교

- `Number.EPSILON`은 자바스크립트에서 표현할 수 있는 **가장 작은 수**입니다.
- 두 값의 차이가 `Number.EPSILON`보다 작으면 같다고 판단하는 방식으로 비교할 수 있습니다.

```js
function isEqual(a, b) {
  return Math.abs(a - b) < Number.EPSILON;
}

console.log(isEqual(0.1 + 0.2, 0.3)); // true
```

## 장점

- (`Number.EPSILON` 방식) 문자열 변환 없이 **부동소수점 오차를 고려한 안전한 비교**가 가능합니다.

## 단점

- (반올림 방식) `toFixed()`, `toPrecision()` 사용 시 **결과가 문자열**이 되어 후속 계산에서 추가 변환이 필요할 수 있습니다.

## 주의사항 및 실무 팁

- 부동소수점 비교는 가능한 한 `===`로 직접 비교하기보다, **허용 오차(ε)를 둔 비교**를 고려하세요.
- 반올림 방식은 화면 표시(포맷팅) 목적에는 유용하지만, **계산 로직에서는 타입 변환 부작용**을 주의해야 합니다.
