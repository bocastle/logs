# React Portal 정리

## 핵심 요약

- `React Portal`은 컴포넌트의 렌더링 위치를 DOM 트리의 다른 곳으로 보낼 수 있게 해 주는 기능이다.
- 핵심은 `React 트리와 DOM 트리는 꼭 같은 위치일 필요가 없다`는 점이다.
- 면접에서는 모달 구현 기술 정도로만 말하지 말고, `이벤트 전파와 상태 관리는 여전히 React 트리를 기준으로 동작한다`는 점까지 같이 설명하면 좋다.
- 주로 모달, 툴팁, 드롭다운처럼 부모의 `overflow`, `z-index` 영향에서 벗어나야 하는 UI에서 많이 쓴다.

## React Portal이 뭐냐

일반적으로 React 컴포넌트는 부모 아래에 렌더링된다.  
그런데 모달이나 툴팁처럼 화면 최상단에 떠야 하는 UI는 부모 DOM 구조 안에 그대로 들어가면 레이아웃 제약을 받을 때가 많다.

이럴 때 Portal을 쓰면 `컴포넌트는 기존 React 트리에 두되, 실제 DOM만 다른 위치에 렌더링`할 수 있다.

예를 들어 `#modal-root` 같은 별도 DOM 노드에 모달을 그리는 식이다.

## 왜 필요한가

다음 같은 상황에서 Portal이 특히 유용하다.

- 부모 요소에 `overflow: hidden`이 걸려 있어 UI가 잘리는 경우
- `z-index` 계층이 꼬여서 팝업이 가려지는 경우
- 레이아웃 구조는 유지하고 싶지만, 렌더링 위치는 화면 최상단이어야 하는 경우

즉 `상태와 로직은 현재 컴포넌트 문맥을 유지하면서, 화면 출력 위치만 분리하고 싶을 때` 쓰는 기능이다.

## 어떻게 동작하나

보통은 `createPortal`을 사용한다.

```tsx
return createPortal(<Modal />, document.getElementById('modal-root'));
```

이렇게 하면 `Modal`은 DOM상으로는 `modal-root` 아래에 렌더링되지만, React 관점에서는 원래 부모 컴포넌트의 자식으로 취급된다.

## 이벤트 전파는 어떻게 되나

이 부분이 면접 포인트다.

Portal로 렌더링됐다고 해서 React 이벤트 흐름이 DOM 위치 기준으로만 바뀌는 건 아니다.  
React의 이벤트 전파는 기본적으로 React 트리를 기준으로 동작한다.

즉 모달이 DOM상으로 바깥에 있어도, 상위 React 컴포넌트의 핸들러가 이벤트를 받을 수 있다.

이걸 알고 있으면 "DOM은 바깥에 있지만 상태와 이벤트 문맥은 이어진다"라고 정리할 수 있다.

## 모달 말고 어디에 쓰나

- 모달
- 툴팁
- 드롭다운 메뉴
- 토스트 알림
- 컨텍스트 메뉴

공통점은 `시각적으로는 상위 레이어에 떠야 하지만, 로직은 현재 컴포넌트와 연결되어야 하는 UI`라는 점이다.

## 주의할 점은 뭐냐

Portal이 만능은 아니다.

- 접근성 처리를 따로 신경 써야 한다.
- 포커스 이동, ESC 닫기, 배경 스크롤 잠금 같은 동작은 Portal만으로 해결되지 않는다.
- DOM 위치가 분리되기 때문에 스타일 범위나 테스트 셀렉터를 놓치기 쉽다.

특히 모달 면접 질문에서는 `Portal을 쓴다`에서 끝내지 말고, `포커스 트랩과 접근성까지 고려해야 한다`고 이어 가면 좋다.

## 면접 답변 예시

> React Portal은 컴포넌트를 현재 React 트리 바깥의 다른 DOM 노드에 렌더링할 수 있게 해 주는 기능입니다. 주로 모달이나 툴팁처럼 부모의 overflow나 z-index 영향을 피해야 하는 UI에서 사용합니다. 다만 DOM 위치만 바뀌는 것이고, 상태 관리나 React 이벤트 전파는 기존 React 트리를 기준으로 유지된다는 점이 중요합니다. 그래서 단순히 화면에 띄우는 것뿐 아니라 접근성, 포커스 제어까지 함께 고려해야 합니다.

## 장점

- 모달 구현 경험을 구조적으로 설명하기 좋다.
- React 트리와 DOM 트리의 차이를 이해하고 있다는 점을 보여 주기 좋다.
- 접근성과 UI 레이어 설계 이야기까지 확장하기 쉽다.

## 단점

- 단순히 "모달 만들 때 쓰는 것"으로만 답하면 깊이가 부족해 보인다.
- 이벤트 전파나 포커스 제어를 모르면 실무 경험이 얕아 보일 수 있다.
- CSS 문제 해결책처럼만 말하면 React 개념 이해가 약해 보일 수 있다.

## 주의사항 / 실무 팁

- 면접에서는 `React 트리와 DOM 위치를 분리한다`는 한 줄 정의를 먼저 말하면 좋다.
- 모달 예시를 들되, 왜 일반 렌더링으로는 불편한지 같이 설명하면 더 자연스럽다.
- 이벤트는 React 트리 기준으로 전파된다는 점을 기억해 두면 후속 질문 대응이 쉽다.
- 실무 팁으로는 포커스 트랩, aria 속성, 배경 스크롤 잠금을 함께 언급하면 좋다.
