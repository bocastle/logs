# CSS has()로 상태 기반 스타일링할 때 주의점 정리

## 핵심 요약

- 상태 class를 JS로 덧붙이는 코드를 줄인다.
- 넓은 범위의 `:has()`는 스타일 계산 비용을 키울 수 있다.
- 가까운 wrapper 기준으로 선택 범위를 좁힌다.

## 개념 설명

`:has()`는 특정 자식이나 후속 조건을 가진 요소를 부모 쪽에서 선택할 수 있게 하는 관계형 CSS 선택자다.

폼 안에 invalid input이 있을 때 fieldset을 강조하거나, 카드 안 이미지 유무에 따라 레이아웃을 바꾸는 식으로 상태를 위로 끌어올린다.

## 예시

```css
.form-field:has(input:invalid) {
  border-color: var(--danger);
}
```

필드 내부 input 상태로 바깥 wrapper를 스타일링한다. 복잡한 전역 선택자에 쓰면 스타일 재계산 범위가 커질 수 있다.

## 면접 답변 예시

> 폼 오류와 카드 변형을 CSS만으로 표현하기 쉽다. 복잡한 조건은 CSS보다 상태 class가 더 읽기 쉬울 수 있다. Performance 패널에서 style recalculation 변화를 확인한다.

## 장점

- 부모 컨테이너 스타일을 실제 DOM 상태와 연결한다.

## 단점

- 지원 브라우저를 확인해야 한다.

## 주의사항 / 실무 팁

- 폼 상태, 카드 변형처럼 관계가 분명한 곳에 쓴다.
