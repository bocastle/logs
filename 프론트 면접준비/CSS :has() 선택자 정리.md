# CSS :has() 선택자 정리

## 핵심 요약
- `:has()`는 부모나 형제 조건을 기준으로 요소를 선택할 수 있게 해 주는 CSS 선택자다.
- 예전에는 자바스크립트로 처리하던 UI 상태 표현 일부를 CSS만으로 풀 수 있다는 점이 핵심이다.
- 면접에서는 "하위 요소 상태를 보고 상위 요소 스타일을 바꿀 수 있게 해 주는 관계형 선택자"라고 설명하면 된다.
- 다만 강력한 만큼 선택 범위와 성능 영향을 고려해서 써야 한다.

## 개념 설명
기존 CSS는 주로 "현재 요소"나 "자식 요소"를 기준으로 스타일을 적용하는 데 강했다. 반대로 "이 부모 안에 특정 자식이 있으면 부모를 바꾸고 싶다" 같은 요구는 상대적으로 어려웠다.

` :has()`는 이런 한계를 줄여 준다. 예를 들어 어떤 카드 안에 에러 메시지가 있으면 카드 테두리를 빨갛게 바꾸거나, 체크된 체크박스를 포함한 라벨만 강조하는 식의 패턴을 CSS만으로 표현할 수 있다.

이 선택자는 흔히 "부모 선택자처럼 보인다"라고 소개되지만, 정확히는 조건부 관계형 선택자에 가깝다. 즉, 특정 하위 구조나 상태를 만족하는 요소를 선택하는 방식이다.

핵심은 다음과 같다.

- 자식이나 후손 조건을 보고 현재 요소를 선택할 수 있다
- 단순 상태 표현은 JS 없이 CSS만으로 구현 가능해진다
- 강력하지만 과도하게 넓은 범위에서 쓰면 유지보수와 성능이 나빠질 수 있다

## 예시
예를 들어 폼 필드 묶음이 있고, 내부 input이 invalid 상태면 전체 래퍼를 강조하고 싶다고 해 보자.

```css
.field:has(input:invalid) {
  border-color: red;
}
```

이전에는 input 상태를 감지해 부모에 클래스를 붙이는 자바스크립트가 필요했지만, 이제는 CSS만으로도 충분한 경우가 생긴다.

또 메뉴 항목 안에 활성 링크가 있으면 상위 메뉴를 강조하는 식의 UI도 표현하기 쉽다.

## 면접 답변 예시
- 질문: "`:has()` 선택자는 무엇인가요?"  
  답변: "하위 요소의 존재나 상태를 기준으로 현재 요소를 선택할 수 있게 해 주는 CSS 선택자입니다. 그래서 예전에는 자바스크립트로 부모에 클래스를 붙여야 하던 UI 일부를 CSS만으로 처리할 수 있습니다."
- 질문: "실무에서 왜 유용한가요?"  
  답변: "폼 검증, 활성 상태 강조, 카드 내부 상태에 따른 스타일 변경처럼 구조적 관계가 중요한 UI에서 유용합니다. 다만 선택 범위를 너무 넓게 잡으면 복잡해질 수 있어서 필요한 범위에만 쓰는 게 좋습니다."

## 장점
- 자바스크립트 없이도 표현 가능한 UI 상태가 늘어난다
- 마크업 구조를 활용한 선언적 스타일링이 쉬워진다
- 폼 검증, 선택 상태, 활성 메뉴 등 실무 예시를 말하기 좋다

## 단점
- 브라우저 지원 상황을 확인해야 한다
- 복잡한 선택자는 가독성과 유지보수성을 해칠 수 있다
- 넓은 DOM 범위에서 남용하면 성능 부담을 만들 수 있다

## 주의사항 / 실무 팁
- `:has()`는 강력해서 오히려 CSS 로직을 과도하게 복잡하게 만들 수 있다. "읽기 쉬운가"를 같이 봐야 한다.
- 상태 표현이 단순하면 유용하지만, 비즈니스 로직까지 CSS로 밀어 넣는 식의 사용은 피하는 편이 좋다.
- 지원 범위가 문제 되는 환경에서는 점진적 향상 관점으로 접근하는 것이 안전하다.
- 면접에서는 "`부모를 선택한다`보다 `하위 상태를 기반으로 현재 요소를 스타일링한다`"라고 설명하면 더 정확하다.
