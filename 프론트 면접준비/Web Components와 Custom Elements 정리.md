# Web Components와 Custom Elements 정리

## 핵심 요약
- `Web Components`는 프레임워크에 종속되지 않는 재사용 UI 컴포넌트를 만들기 위한 웹 표준 묶음이다.
- `Custom Elements`는 그중에서 개발자가 직접 새로운 HTML 태그를 정의하는 기능이다.
- 면접에서는 "브라우저 표준 기반 컴포넌트 모델이며, Custom Elements는 사용자 정의 태그를 만드는 축"이라고 말하면 핵심이 빠르게 전달된다.
- `Shadow DOM`, 템플릿 재사용, 프레임워크 독립성까지 연결해서 설명하면 답변 밀도가 좋아진다.

## 개념 설명
프론트엔드에서 컴포넌트라는 개념은 보통 React, Vue 같은 프레임워크를 통해 먼저 접한다. 그런데 브라우저 자체도 컴포넌트를 만들 수 있는 표준을 제공한다. 그 표준 묶음이 `Web Components`다.

보통 Web Components를 이야기할 때는 세 가지를 같이 본다.

- `Custom Elements`: 새로운 HTML 태그를 정의하는 기능
- `Shadow DOM`: 내부 DOM과 스타일을 캡슐화하는 기능
- `HTML Template`: 재사용할 마크업 조각을 선언하는 기능

이 중 면접에서 가장 먼저 나오는 키워드는 `Custom Elements`다. 예를 들어 `<user-card>` 같은 태그를 직접 정의하고, 그 안의 동작과 생명주기를 자바스크립트 클래스로 관리할 수 있다.

즉, Web Components는 "브라우저 표준으로 컴포넌트를 만드는 방식"이고, Custom Elements는 그 안에서 "직접 태그를 정의하는 기능"이라고 이해하면 된다.

## 예시
사내 여러 프로젝트에서 공통 버튼, 모달, 배지 같은 UI를 써야 한다고 해 보자.

- 특정 프로젝트는 React를 쓰고
- 다른 프로젝트는 서버 템플릿이나 Vue를 쓸 수 있다
- 이때 프레임워크마다 별도 구현을 유지하면 비용이 커진다

이런 상황에서 Web Components로 공통 UI를 만들면 브라우저가 이해하는 표준 태그 형태로 재사용할 수 있다. 예를 들어 `<app-modal>`을 정의해 두면, 어느 프론트엔드 스택이든 HTML처럼 붙여 사용할 수 있다.

물론 실제로는 상태 관리, SSR, 폼 처리 같은 부분에서 프레임워크와의 결합 문제가 생길 수 있으니 무조건 정답은 아니다.

## 면접 답변 예시
- 질문: "`Web Components`는 무엇인가요?"  
  답변: "브라우저 표준으로 재사용 가능한 UI 컴포넌트를 만드는 방식입니다. 보통 Custom Elements, Shadow DOM, Template를 함께 묶어서 말합니다."
- 질문: "`Custom Elements`는 무엇인가요?"  
  답변: "개발자가 새로운 HTML 태그를 직접 정의하는 기능입니다. 예를 들어 `<user-card>` 같은 태그를 만들고, 생명주기와 동작을 자바스크립트 클래스로 연결할 수 있습니다."
- 질문: "언제 고려할 수 있나요?"  
  답변: "여러 프레임워크나 여러 제품군에서 공통 UI를 공유해야 할 때 검토할 수 있습니다. 브라우저 표준 기반이라 프레임워크 독립성이 장점이 되기 때문입니다."

## 장점
- 특정 프레임워크에 종속되지 않는 공통 컴포넌트를 만들 수 있다
- 브라우저 표준 기반이라 장기적으로 기술 스택이 바뀌어도 재사용 여지가 있다
- `Shadow DOM`과 함께 쓰면 스타일 충돌을 줄이기 좋다

## 단점
- React 같은 프레임워크의 상태 관리 방식과 자연스럽게 맞지 않을 수 있다
- 브라우저 표준이라고 해도 실제 개발 경험은 프레임워크 컴포넌트보다 불편할 수 있다
- SSR, 폼 연동, 이벤트 처리에서 추가 래핑이나 규약이 필요할 수 있다

## 주의사항 / 실무 팁
- "표준이니까 무조건 좋다"는 식으로 말하면 약하다. 왜 프레임워크 컴포넌트 대신 이걸 써야 하는지 맥락이 있어야 한다.
- 공통 디자인 시스템을 여러 앱에서 공유하는 상황이면 후보가 되지만, 단일 React 앱이라면 React 컴포넌트만으로도 충분한 경우가 많다.
- `Custom Elements`와 `Shadow DOM`을 구분해서 말하면 이해도가 드러난다. 전자는 태그 정의, 후자는 캡슐화다.
- 면접에서는 "`Web Components는 브라우저 표준 컴포넌트 모델이고, Custom Elements는 그중 사용자 정의 태그를 만드는 기능`"이라고 먼저 정리하면 안정적으로 답할 수 있다.
