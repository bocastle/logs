# 자바스크립트에서 메모리 누수가 발생하는 상황 정리

## 핵심 요약

- 메모리 누수는 개발자가 **의도치 않게 참조를 유지**해 가비지 컬렉터가 메모리를 회수하지 못할 때 발생합니다.
- 대표 원인: **이벤트 리스너 해제 누락**, **클로저로 인한 불필요한 참조 유지**, **전역 변수 남용**.
- 크롬 개발자 도구 **Memory 탭**으로 힙 상태와 할당/해제 흐름을 모니터링할 수 있습니다.

## 메모리 누수가 발생하는 대표 상황

### 이벤트 리스너를 해제하지 않는 경우

특정 DOM 요소에 이벤트 리스너를 등록한 뒤 해당 요소를 제거할 때, 이벤트 리스너를 명시적으로 제거하지 않으면 해당 요소에 대한 참조가 유지되어 메모리가 해제되지 않을 수 있습니다.

- 방지 방법: 불필요한 이벤트 리스너를 `removeEventListener()`로 제거

```js
const handler = () => {
  // ...
};

element.addEventListener("click", handler);

// element를 제거하기 전에 반드시 해제
element.removeEventListener("click", handler);
```

### 클로저를 잘못 사용하는 경우

클로저는 내부 함수가 외부 함수의 변수를 참조할 수 있게 해주지만, 특정 변수나 DOM 요소를 참조하는 클로저가 계속 유지되면 가비지 컬렉터가 해당 메모리를 회수하지 못할 수 있습니다.

```js
function createLeakyHandler(dom) {
  const bigData = new Array(1000000).fill("x");

  // dom과 bigData를 참조하는 클로저가 오래 살아남으면 누수로 이어질 수 있음
  return function onClick() {
    console.log(dom, bigData.length);
  };
}
```

### 전역 변수를 과도하게 사용하는 경우

전역 객체(`window`)에 저장된 변수는 애플리케이션이 종료될 때까지 유지되므로, 필요하지 않은 데이터를 전역 변수로 두면 메모리 누수가 발생할 수 있습니다.

- 방지 방법: 전역 범위를 최소화하고, 필요하면 즉시 실행 함수(IIFE)나 모듈 패턴으로 관리

```js
// 전역 남용 (지양)
window.cache = {
  /* ... */
};

// 전역 범위를 줄이는 예시 (IIFE)
(() => {
  const cache = {
    /* ... */
  };
  // 필요한 곳에서만 사용
})();
```

## 자바스크립트에서 메모리 사용량을 모니터링하는 방법

크롬 개발자 도구의 **Memory 탭**을 활용하면 메모리 사용량을 확인할 수 있습니다.

### Heap snapshot

현재 힙 메모리에 어떤 객체들이 존재하는지 파악할 수 있습니다.

### Allocations on timeline

시간에 따른 메모리 할당 및 해제 상태를 실시간으로 확인할 수 있습니다.

## 장점

- 원문에서는 별도의 장점이 직접적으로 정리되어 있지는 않았습니다.

## 단점

- 원문에서는 별도의 단점이 직접적으로 정리되어 있지는 않았습니다.

## 주의사항 / 실무 팁

- DOM을 제거하기 전에, 해당 DOM에 걸린 이벤트 리스너가 남아있는지(또는 상위에서 참조가 유지되는지) 점검하세요.
- 클로저가 큰 객체/DOM 참조를 오래 잡고 있지 않도록, 생명주기(마운트/언마운트, 구독/해제)를 명확히 관리하세요.
- 전역(`window`)에 데이터를 올려두는 패턴은 최소화하고, 필요한 경우에도 “언제 해제되는지”가 보이도록 구조화하세요.
- 누수가 의심되면 크롬 DevTools **Memory**에서 `Heap snapshot`, `Allocations on timeline`으로 원인을 추적해 보세요.
