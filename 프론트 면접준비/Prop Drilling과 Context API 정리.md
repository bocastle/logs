# Prop Drilling과 Context API 정리

## 핵심 요약

- `Prop Drilling`은 실제로 쓰지 않는 중간 컴포넌트까지 props를 계속 전달해야 하는 상황을 말한다.
- `Context API`는 여러 단계 아래 컴포넌트에 공통 데이터를 전달할 때 중간 전달을 줄이도록 도와주는 React 기능이다.
- 면접에서는 "`Context`가 전역 상태 관리 라이브러리의 완전한 대체"라고 말하기보다, `공유 범위가 분명한 값 전달 수단`으로 설명하는 게 안전하다.
- 작은 공유 상태에는 편리하지만, 값이 자주 바뀌거나 구조가 커지면 리렌더링 관리와 관심사 분리가 중요해진다.

## Prop Drilling이 왜 문제냐

컴포넌트 구조가 깊어질수록 상위에서 하위로 데이터를 전달하려면 중간 단계도 props를 받아서 다시 넘겨야 한다.

예를 들어 `App -> Layout -> Sidebar -> ProfileCard` 구조에서 실제로 사용자 정보가 필요한 건 `ProfileCard`뿐인데, 중간 컴포넌트도 전부 전달 통로가 된다.  
이러면 코드가 지저분해지고, 구조 변경에도 취약해진다.

즉 Prop Drilling 문제는 "props를 쓴다"가 아니라, `중간 계층이 불필요하게 데이터 운반만 하게 되는 상황`에 있다.

## Context API는 어떤 문제를 풀어주나

Context는 특정 범위 아래의 컴포넌트들이 공통으로 필요한 값을 직접 꺼내 쓸 수 있게 해 준다.

대표적으로 로그인 사용자 정보, 테마, 언어 설정처럼 여러 곳에서 필요하지만 앱 전체에 흩뿌릴 필요는 없는 값에 잘 맞는다.

그래서 Context는 깊은 props 전달을 줄이고, "이 값은 이 트리 범위에서 공유된다"는 의도를 드러내는 데 유용하다.

## 그렇다고 Context가 만능은 아니다

Context에 값을 넣으면 해당 값을 구독하는 컴포넌트들이 변경 시 다시 렌더링될 수 있다.  
그래서 값이 자주 바뀌는 큰 상태를 아무 생각 없이 몰아 넣으면 성능과 구조가 금방 복잡해진다.

면접에서는 "단순 공유값에는 좋지만, 복잡한 상태 관리 전체를 대체한다고 보긴 어렵다"라고 말하면 균형이 있다.

## 언제 쓰면 좋나

### Context가 잘 맞는 경우

- 현재 로그인 사용자
- 다크 모드 같은 테마 설정
- 다국어 설정
- 폼이나 탭처럼 특정 하위 트리에서만 공유하는 상태

### Context만으로 버거워질 수 있는 경우

- 값 변경이 아주 잦은 상태
- 비동기 캐시, 서버 상태 관리
- 액션과 파생 상태가 많아지는 큰 전역 상태

이 경우는 상태를 더 잘게 나누거나, 별도 상태 관리 도구를 고려할 수 있다.

## 면접에서 어떻게 비교하면 좋나

Prop Drilling은 단순하고 명시적이라는 장점이 있다.  
그래서 전달 단계가 짧고 구조가 단순하면 굳이 Context를 쓰지 않아도 된다.

반면 여러 단계 아래까지 같은 데이터를 반복 전달해야 한다면 Context가 유지보수에 더 유리하다.  
결국 기준은 "깊이"보다 `중간 컴포넌트가 그 값을 실제로 쓰는지`다.

## 면접 답변 예시

> Prop Drilling은 실제로 데이터가 필요하지 않은 중간 컴포넌트까지 props를 계속 전달해야 하는 상황을 말합니다. React Context API는 이런 전달 부담을 줄여서 특정 트리 아래 컴포넌트들이 공통 값을 직접 사용할 수 있게 해 줍니다. 예를 들어 테마나 로그인 사용자 정보처럼 여러 곳에서 필요하지만 범위가 분명한 값에 잘 맞습니다. 다만 Context가 전역 상태 관리의 만능 대체는 아니고, 값 변경이 잦거나 상태가 복잡해지면 리렌더링과 구조 분리를 더 신경 써야 합니다.

## 장점

- Prop Drilling과 Context의 역할 차이를 말하기 쉬워 기본 React 이해도를 보여 주기 좋다.
- 실무 예시로 테마, 사용자 정보, 언어 설정을 붙이기 쉽다.
- "언제는 그냥 props가 낫고, 언제는 Context가 낫다"는 판단 기준을 말할 수 있다.

## 단점

- Context를 전역 상태 관리 전부로 과장하면 답변이 부정확해질 수 있다.
- Prop Drilling을 무조건 나쁜 패턴처럼 말하면 균형이 깨진다.
- 리렌더링 특성을 모르면 실무 관점이 약하게 들릴 수 있다.

## 주의사항 / 실무 팁

- 면접에서는 Prop Drilling을 "깊게 전달되는 불필요한 데이터 운반"으로 정의하면 깔끔하다.
- Context 예시는 테마나 인증 정보처럼 공감하기 쉬운 케이스로 드는 편이 좋다.
- "props가 명시적이라 더 단순할 때도 있다"는 말을 덧붙이면 과한 일반화를 피할 수 있다.
- 값이 자주 바뀌는 큰 상태는 Context를 분리하거나 다른 도구와 함께 본다고 말하면 실무 감각이 난다.
