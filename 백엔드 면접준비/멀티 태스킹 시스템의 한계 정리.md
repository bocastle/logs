# 멀티 태스킹 시스템의 한계 정리

## 핵심 요약

멀티 태스킹 시스템(프로세스 단위로 여러 작업을 번갈아 수행)은 **동시성의 착시**를 제공하지만, 프로세스 기반 구조만으로는 **자원 소모/전환 비용/데이터 공유** 측면의 한계가 남습니다. 이를 보완하기 위해 **스레드(Thread)** 개념이 등장했습니다.

## 멀티 태스킹 시스템의 한계

멀티 태스킹 시스템을 사용하더라도 다음과 같은 문제점이 있습니다.

### 1) 하나의 프로세스가 동시에 여러 작업을 수행하지 못함

- 여러 프로세스를 생성해 문제를 해결할 수는 있지만,
- 프로세스가 많아질수록 **관리 부담**과 **자원 소모**가 커집니다.

### 2) 무거운 프로세스 간 컨텍스트 스위칭

- 컨텍스트 스위칭은 CPU가 한 프로세스에서 다른 프로세스로 전환할 때 발생합니다.
- 이 작업은 상대적으로 **무겁고 비용이 큽니다.**

### 3) 프로세스 간 데이터 공유의 어려움

- 각 프로세스는 **독립적인 메모리 공간**을 사용합니다.
- 따라서 서로 다른 프로세스 사이에서 **데이터 공유가 까다롭습니다.**

## 스레드(Thread)의 등장과 특징

위 한계를 해결하기 위해 **스레드**가 등장했습니다. 스레드는 **한 프로세스 내에서 여러 작업을 동시에 실행**할 수 있도록 도와줍니다.

### 1) 프로세스 내 여러 스레드 보유

- 하나의 프로세스는 **하나 이상의 스레드**를 가질 수 있고,
- 각 스레드가 하나의 작업을 담당함으로써 여러 작업을 동시에 실행할 수 있습니다.

### 2) CPU 실행 단위

- 과거에는 **프로세스**가 CPU 실행 단위였다면,
- 현재는 **스레드**가 CPU에서 실행되는 **최소 단위**가 되었습니다.
- 기본적으로 프로세스는 하나의 스레드를 가지고, 필요에 따라 추가 스레드를 생성합니다.

### 3) 가벼운 스레드 간 컨텍스트 스위칭

- 스레드는 같은 프로세스 내에서 메모리 영역(특히 **Heap**)을 공유합니다.
- 그래서 스레드 간 컨텍스트 스위칭은 **프로세스 간 스위칭보다 훨씬 가볍습니다.**
- 다만 각 스레드는 고유한 **Stack, 포인터, 프로그램 카운터** 등을 가져 실행 상태를 유지합니다.

## 멀티스레딩과 멀티프로세싱

### 멀티스레딩(Multi-threading)

- 하나의 프로세스 안에서 여러 스레드를 통해 동시에 여러 작업을 실행하는 기법입니다.

### 확장된 멀티태스킹 개념

- 과거에는 프로세스 간 매우 짧은 시간의 스위칭으로 작업을 나눴다면,
- 이제는 여러 프로세스와 스레드가 **아주 짧게 쪼개진 CPU 타임**을 나눠 갖고 실행됩니다.

### 멀티프로세싱(Multi-processing)

- 두 개 이상의 프로세서(또는 코어)를 활용해, 여러 프로세스가 동시에 실행되는 시스템입니다.

## 장점

### 스레드 도입(멀티스레딩)의 장점

- 같은 프로세스 내에서 메모리를 공유하므로 **프로세스 간 공유보다 데이터 공유가 수월**해집니다.
- 스레드 간 컨텍스트 스위칭이 상대적으로 **가볍습니다.**

## 단점

### 멀티프로세스 확장(프로세스만 늘리는 방식)의 단점

- 프로세스 수가 늘수록 **관리 복잡도**와 **자원 소모**가 커집니다.
- 프로세스 간 컨텍스트 스위칭 비용이 **크게 증가**할 수 있습니다.
- 프로세스 간 데이터 공유가 **어렵습니다.**

## 주의사항 / 실무 팁

- “멀티 태스킹”은 보통 **동시 실행처럼 보이게 만드는 스케줄링**까지 포함해 말하는 경우가 많습니다. 면접/설명에서는 **(1) 프로세스 기반 동시성의 한계 → (2) 스레드로 보완** 흐름으로 답변하면 구조가 깔끔합니다.
- “스레드는 가볍다”는 표현을 쓸 때는, **메모리 공유(Heap)로 인해 전환 비용이 상대적으로 낮다**는 근거와 함께, 스레드도 **Stack/PC 등 고유 상태를 가진다**는 점을 같이 언급하면 정확도가 올라갑니다.
