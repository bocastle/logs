# 단일 프로세스 시스템 정리

## 핵심 요약

- **단일 프로세스 시스템**은 **한 번에 하나의 프로그램(프로세스)만 실행**하는 시스템입니다.
- CPU가 IO를 기다리는 동안 **CPU 유휴 시간이 늘어** 효율이 떨어질 수 있고, 이를 보완하기 위해 **멀티 프로그래밍 → 멀티 태스킹**으로 발전할 수 있습니다.

## 단일 프로세스 시스템이란

단일 프로세스 시스템은 **한 번에 하나의 프로그램만 실행**합니다.  
또 다른 프로그램을 실행하려면 **먼저 실행 중이던 프로그램을 종료**한 뒤, 다음 프로그램을 실행해야 합니다.

## 단일 프로세스의 단점과 개선 방법

### 단점

- **CPU 사용률이 좋지 않음**
  - 프로세스가 CPU 작업을 처리하다가 **IO 작업**(디스크/네트워크 등)을 수행하면, 그동안 CPU는 **아무 일도 하지 않고 대기**하게 됩니다.

### 개선: 멀티 프로그래밍(Multi-Programming)

- 한 프로세스가 IO로 대기 상태가 되면 **다른 프로세스를 실행**해
  **CPU가 놀고 있는 시간**을 줄입니다.
- 주된 목적: **CPU 사용률 극대화**

## 멀티 프로그래밍의 단점과 개선 방법

### 단점

- 어떤 프로세스가 **CPU를 계속 점유**(CPU-bound)하면,
  다른 프로세스가 작업을 못 하고 **대기**하는 문제가 생길 수 있습니다.

### 개선: 멀티 태스킹(Multi-Tasking)

- 두 개 이상의 프로세스가 CPU를 사용할 때,
  **아주 짧은 시간(Quantum, Time Slice)** 단위로 **번갈아 실행**되도록 합니다.
- 결과적으로 한 프로세스의 **독점**이나 **긴 대기**로 인한 비효율을 줄입니다.

## 멀티 프로그래밍과 멀티 태스킹의 차이

- 공통점: 동시에 여러 프로그램을 실행시키는 것처럼 동작
- 차이점:
  - **멀티 태스킹**은 CPU 시간을 **아주 짧게 쪼개** 프로세스들이
    **교대로 실행**되도록 한다는 점이 핵심입니다.
- 멀티 태스킹의 목적:
  - **프로세스의 응답 시간 최소화**
  - 사용자 입장에서는 **정말 여러 프로그램이 동시에 실행되는 것처럼** 느껴지게 함
  - 짧은 타임 슬라이스 안에서 교대 실행하며 **즉각적인 반응**을 제공

## 장점

### 단일 프로세스 시스템의 장점

- 구조가 단순하고, 한 번에 하나만 실행하므로 **개념적으로 이해/구현이 쉬움**  
  (다만, 실사용 관점에서는 단점이 더 크게 체감되는 경우가 많습니다.)

## 단점

### 단일 프로세스 시스템의 단점

- IO 대기 동안 CPU가 놀게 되어 **CPU 사용률이 낮아짐**
- 여러 작업을 동시에 처리하기 어렵고, 프로그램 전환에 **종료/재실행**이 필요

## 주의사항 / 실무 팁

- “멀티 프로그래밍”과 “멀티 태스킹”을 혼동하기 쉬우니 면접에서는 아래처럼 구분해 설명하면 좋습니다.
  - 멀티 프로그래밍: **CPU 유휴 시간 감소(사용률 극대화)** 중심
  - 멀티 태스킹: **타임 슬라이스(Quantum) 기반으로 응답성(사용자 체감) 향상** 중심
- Quantum(타임 슬라이스)을 너무 짧게 잡으면 문맥 전환(Context Switch) 비용이 늘 수 있고,
  너무 길면 응답성이 떨어질 수 있다는 점도 함께 언급하면 좋습니다.
