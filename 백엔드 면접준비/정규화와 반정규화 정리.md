# 정규화와 반정규화 정리

## 핵심 요약

- `정규화`는 데이터 중복을 줄이고 이상 현상을 막기 위해 테이블 구조를 더 잘게 나누는 과정이다.
- `반정규화`는 조회 성능이나 사용 편의를 위해 일부 중복을 감수하고 구조를 다시 합치거나 컬럼을 추가하는 접근이다.
- 면접에서는 "정규화가 무조건 좋다"가 아니라, `데이터 무결성`과 `조회 성능` 사이의 균형 문제로 설명하는 게 좋다.
- 실무에서는 처음부터 과하게 반정규화하기보다, 정규화된 구조를 기본으로 두고 병목이 확인된 뒤 필요한 부분만 조정하는 편이 안전하다.

## 정규화는 왜 하냐

정규화는 같은 데이터를 여러 군데에 반복 저장하지 않게 만들어서 수정 비용을 줄이고, 데이터가 서로 다르게 꼬이는 일을 막기 위해 한다.

예를 들어 주문 테이블마다 사용자 이름과 주소를 계속 복사해 넣으면, 주소가 바뀔 때 여러 행을 다 수정해야 한다.  
이런 구조는 수정 누락이 생기기 쉽고, 결국 데이터 신뢰도가 떨어진다.

그래서 정규화는 "한 사실은 한 곳에만 저장하자"는 방향으로 이해하면 된다.

## 이상 현상이란 뭐냐

면접에서 자주 같이 나오는 표현이 이상 현상이다. 대표적으로 세 가지가 있다.

- 삽입 이상: 아직 주문이 없으면 사용자 정보를 넣을 수 없는 식의 문제
- 수정 이상: 같은 정보가 여러 행에 있어서 일부만 바뀌는 문제
- 삭제 이상: 주문 한 건을 지웠더니 사용자 정보까지 사라지는 문제

정규화는 이런 문제를 줄이기 위한 데이터 모델링 기법이다.

## 반정규화는 왜 필요하냐

정규화만 밀어붙이면 조회할 때 조인이 많아질 수 있다.  
특히 읽기 비중이 높고, 자주 함께 조회되는 데이터가 뚜렷하면 성능이나 쿼리 복잡도가 문제가 될 수 있다.

이럴 때는 일부 컬럼을 중복 저장하거나, 집계 값을 따로 들고 있거나, 조회 전용 테이블을 두는 식으로 반정규화를 고려한다.

즉 반정규화는 잘못된 설계가 아니라, 조회 요구사항이 강할 때 의도적으로 택하는 최적화 전략이다.

## 정규화와 반정규화를 어떻게 비교하면 좋나

### 정규화의 강점

정규화는 데이터 변경이 일관되고 무결성을 지키기 좋다.  
쓰기 로직이 명확하고, 중복 데이터로 인한 버그를 줄이는 데 유리하다.

### 반정규화의 강점

반정규화는 읽기 성능과 조회 편의성에 강하다.  
특정 화면이나 API에서 필요한 데이터를 한 번에 가져오기 쉬워질 수 있다.

면접에서는 "쓰기 중심이면 정규화, 읽기 병목이 분명하면 선택적으로 반정규화" 정도로 정리하면 흐름이 깔끔하다.

## 실무에서 자주 나오는 예시

예를 들어 게시글 목록 화면에서 댓글 수를 매번 `COUNT`로 계산하면 비용이 크다고 해 보자.  
이럴 때 게시글 테이블에 댓글 수 컬럼을 따로 두는 건 전형적인 반정규화 사례다.

대신 이 구조를 쓰면 댓글 추가/삭제 시 카운트를 같이 맞춰야 한다.  
즉 조회는 빨라지지만, 쓰기 로직과 동기화 책임이 늘어난다.

## 면접 답변 예시

> 정규화는 데이터 중복을 줄이고 삽입, 수정, 삭제 이상 현상을 방지하기 위해 테이블을 역할별로 분리하는 설계입니다. 반대로 반정규화는 조회 성능이나 사용 패턴을 고려해서 일부 중복을 허용하는 최적화 전략입니다. 저는 기본적으로 정규화를 우선하고, 실제로 조인이 과도하거나 집계 비용이 큰 구간에서만 선택적으로 반정규화를 적용하는 편이 좋다고 봅니다. 결국 둘은 정답과 오답 관계라기보다 무결성과 성능 사이의 트레이드오프라고 생각합니다.

## 장점

- 데이터 모델링 의도를 설명하기 좋아서 면접 답변 구조가 안정적이다.
- 무결성, 조회 성능, 트레이드오프를 한 번에 연결해서 말할 수 있다.
- DB 설계 질문이 나오면 실무 감각을 보여 주기 좋다.

## 단점

- 정규화 단계만 외우면 실제 설계 판단 기준을 설명하기 어렵다.
- 반정규화를 성능 향상 공식처럼 말하면 위험하다.
- 조회 패턴 없이 이론만 말하면 답변이 추상적으로 들릴 수 있다.

## 주의사항 / 실무 팁

- 면접에서는 1정규형, 2정규형을 길게 나열하기보다 "중복 제거와 이상 현상 방지"에 초점을 맞추는 편이 낫다.
- 반정규화는 "조인이 싫어서"가 아니라 "병목이 확인돼서" 선택했다고 말해야 설득력이 있다.
- 집계 컬럼이나 조회 전용 테이블을 예시로 들면 실무적으로 들린다.
- 반정규화 후에는 데이터 동기화 전략까지 같이 말해야 답변 완성도가 올라간다.
