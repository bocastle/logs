# 열 기반 DB와 행 기반 DB의 차이점

## 핵심 요약

- **행 기반 DB(Row-oriented DB)** 는 데이터를 **행 단위**로 관리하며, **행 단위 읽기/쓰기 연산**에 최적화되어 있습니다.
- **열 기반 DB(Column-oriented DB)** 는 데이터를 **열 단위**로 관리하며, 조회 시 **필요한 열만 로드**해 디스크 I/O를 줄이고, **압축 효율**이 높아 주로 **데이터 분석**에 사용됩니다.
- 대표 예시: 행 기반은 **PostgreSQL, MySQL**, 열 기반은 **BigQuery, Redshift, Snowflake** 입니다.

## 행 기반 데이터베이스(Row-oriented Database)

행 기반 데이터베이스는 데이터를 **행 단위로 관리**하는 DBMS입니다.  
행 단위 읽기 및 쓰기 연산에 최적화되어 있습니다.

- 대표적인 행 기반 DB: PostgreSQL, MySQL

## 열 기반 데이터베이스(Column-oriented Database)

열 기반 데이터베이스는 데이터를 **열 단위로 관리**한다는 점에서 행 기반 DB와 차이가 있습니다.

- 데이터를 조회할 때 **필요한 열만 로드**하기 때문에 디스크 I/O를 줄일 수 있습니다.
- 같은 종류의 데이터가 연속적으로 저장되므로 **압축 효율이 높습니다.**
- 이러한 특징으로 인해 주로 **데이터 분석**에 사용됩니다.

- 대표적인 열 기반 DB: BigQuery, Redshift, Snowflake

## 예시로 보는 저장 방식 차이

예를 들어 아래와 같은 데이터가 있다고 가정합니다.

```text
Name        CreatedAt
Atom        2024-01-23
Prin        2024-02-01
Gosmdochee  2024-02-03
```

### 행 기반 DB의 저장 방식

```text
[Atom, 2024-01-23] [Prin, 2024-02-01] [Gosmdochee, 2024-02-03]
```

### 열 기반 DB의 저장 방식

```text
[Atom, Prin, Gosmdochee] [2024-01-23, 2024-02-01, 2024-02-03]
```

## 장점

### 행 기반 DB의 장점

- 행 단위 읽기/쓰기 연산에 최적화되어 있습니다.

### 열 기반 DB의 장점

- 조회 시 필요한 열만 로드해 **디스크 I/O를 줄일 수 있습니다.**
- 같은 종류의 데이터가 연속 저장되어 **압축 효율이 높습니다.**
- 이러한 특성으로 **데이터 분석**에 유리합니다.

## 단점

- 원문에서는 단점이 별도로 직접 언급되지는 않았습니다.

## 주의사항 / 실무 팁

- “트랜잭션 중심(OLTP)”처럼 **행 단위 갱신/조회**가 잦은 워크로드는 보통 **행 기반 DB**가 적합한 경우가 많습니다.
- “분석/집계(OLAP)”처럼 **특정 컬럼 중심 조회/집계**가 많은 워크로드는 보통 **열 기반 DB**가 유리한 경우가 많습니다.
