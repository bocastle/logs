# 멀티 파티셔닝 정리

## 핵심 요약

- 멀티 파티셔닝은 하나의 큰 테이블을 여러 파티션으로 분리해 운영 효율을 높이는 기법이다.
- 쿼리 성능 개선, 운영성, 보존 주기 관리에서 유리하지만 설계 실수가 치명적이다.
- 면접에서는 파티션키 선택 기준과 인덱스/조인 비용을 함께 설명해야 한다.

## 개념 설명

### 1) 파티셔닝의 목적

- 대용량 데이터를 논리적으로 쪼개서 관리
- 오래된 데이터 아카이빙/삭제 운영을 단순화
- 파티션 단위의 작업을 활용해 I/O를 제한

### 2) 파티션 키 설계

- 시간 기반: 주간/월간 단위 관리에 유리
- 키 기반: 샤딩 성격의 접근에 가깝지만 파티션 단위 필터 성능 확보 목적

### 3) 쿼리 영향

- 파티션 프루닝이 되면 조회 범위 축소
- 하지만 비효율한 키는 오히려 모든 파티션 스캔을 유발

## 사용 예시

- 로그성 데이터(시계열)
- 세금/정산 데이터의 월별 보존 관리
- 만료 정책이 명확한 이벤트 테이블

## 면접 답변 예시

"멀티 파티셔닝은 대량 데이터 관리에서 파티션 단위로 I/O를 줄일 수 있어 성능과 운영성을 함께 개선합니다. 하지만 첫 단계는 파티션 키입니다. 접근 패턴이 자주 바뀌면 키가 바뀐 쿼리에 대해서는 역효과가 생길 수 있어, 실제 쿼리 로그 기반으로 설계를 검증해야 합니다. 또한 파티션 간 조인 비용까지 미리 계산해야 장애 없이 운영 가능합니다."

## 장점

- 최근 데이터 조회 성능 개선
- 오래된 데이터 보관/삭제 자동화
- 백업/복구 범위 분리

## 단점

- 설계 잘못 시 파티션 스캔 폭증
- DDL 관리 복잡
- 일부 집계 쿼리 비용 증가 가능

## 주의사항 / 실무 팁

- 시간 기반 파티션과 인덱스 전략을 동시 검토한다.
- 파티션 단위 백업/복구 계획을 문서화한다.
- 파티션 수를 무리하게 늘리지 않는다.
