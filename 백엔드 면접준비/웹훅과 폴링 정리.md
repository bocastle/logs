# 웹훅과 폴링 정리

## 핵심 요약

- `폴링`은 클라이언트가 주기적으로 서버에 상태를 묻는 방식이고, `웹훅`은 이벤트가 생겼을 때 서버가 먼저 콜백을 보내는 방식이다.
- 핵심 차이는 `누가 먼저 통신을 시작하느냐`와 `불필요한 요청이 얼마나 생기느냐`다.
- 면접에서는 단순 비교로 끝내지 말고, `실시간성`, `구현 난이도`, `실패 재전송`, `보안 검증`까지 같이 설명하는 게 좋다.
- 둘 중 하나가 항상 정답은 아니고, 시스템 성격에 따라 선택이 달라진다.

## 폴링이 뭐냐

폴링은 클라이언트가 일정 주기마다 서버에 요청을 보내서 상태 변화를 확인하는 방식이다.

예를 들어 결제 상태 확인 화면에서 3초마다 `결제됐나요?`를 묻는 식이다.

구현은 단순하다.  
클라이언트가 타이머를 돌리면서 같은 API를 반복 호출하면 된다.

## 웹훅이 뭐냐

웹훅은 어떤 이벤트가 발생했을 때 서버가 미리 등록된 URL로 HTTP 요청을 보내 주는 방식이다.

예를 들어 결제 대행사에서 결제 완료 이벤트가 발생하면, 우리 서버의 콜백 URL로 결과를 전달하는 식이다.

즉 폴링이 `내가 계속 물어보는 구조`라면, 웹훅은 `상대가 일이 생기면 알려 주는 구조`다.

## 폴링의 장단점은 뭐냐

장점부터 보면 구현이 쉽고 흐름이 직관적이다.

- 클라이언트나 서버 구조를 크게 바꾸지 않아도 된다.
- 상대 시스템이 웹훅을 지원하지 않아도 동작한다.
- 디버깅이 비교적 쉽다.

하지만 단점도 분명하다.

- 상태가 안 바뀌어도 계속 요청이 나간다.
- 주기가 짧으면 서버 부하가 커진다.
- 주기가 길면 반응이 늦어진다.

즉 `간단하지만 비효율이 생기기 쉬운 방식`이다.

## 웹훅의 장단점은 뭐냐

웹훅은 이벤트가 발생했을 때만 요청이 오니 훨씬 효율적이다.

- 불필요한 요청이 줄어든다.
- 이벤트 전달이 상대적으로 빠르다.
- 외부 시스템 연동에서 비동기 구조를 만들기 좋다.

반면 운영 난이도는 올라간다.

- 콜백 엔드포인트를 외부에 열어야 한다.
- 요청 진위 검증이 필요하다.
- 실패 시 재전송 정책과 중복 처리 전략이 필요하다.

그래서 웹훅은 `효율적이지만 설계와 운영을 더 신경 써야 하는 방식`이라고 정리하면 된다.

## 실무에서는 어떤 점을 조심하나

웹훅은 특히 아래를 조심해야 한다.

- 서명 검증이나 토큰 검증으로 보안 확인
- 재전송 때문에 같은 이벤트가 여러 번 와도 안전한 멱등 처리
- 처리 실패 시 재시도 정책 설계
- 순서 보장이 필요한 이벤트인지 확인

폴링도 마냥 쉬운 건 아니다.

- 주기를 너무 짧게 잡으면 트래픽 낭비가 크다.
- 종료 조건을 잘못 잡으면 무한 요청이 될 수 있다.
- 여러 사용자가 동시에 돌리면 쓸데없는 조회 부하가 커진다.

## 언제 무엇을 선택하나

대체로 이렇게 판단하면 된다.

- 외부 시스템이 웹훅을 지원하고 이벤트 기반 연동이 중요하면 웹훅
- 상대 시스템 제약이 크거나 빠르게 단순 구현해야 하면 폴링
- 사용자 화면에서는 잠깐 폴링하고, 서버 간 최종 정합성은 웹훅으로 맞추는 혼합 방식도 흔하다

예를 들어 결제 완료 UX는 화면에서 짧게 폴링할 수 있지만, 최종 주문 확정은 PG 웹훅 기준으로 처리하는 식이다.

## 면접 답변 예시

> 폴링은 클라이언트가 일정 주기마다 상태를 조회하는 방식이고, 웹훅은 이벤트가 생겼을 때 서버가 등록된 콜백 URL로 먼저 알려 주는 방식입니다. 폴링은 구현이 단순하지만 불필요한 요청이 많아질 수 있고, 웹훅은 효율적이지만 보안 검증과 재전송, 멱등 처리까지 고려해야 합니다. 그래서 저는 외부 시스템이 지원한다면 최종 상태 동기화는 웹훅을 선호하고, 사용자 화면에서 짧은 대기 UX가 필요하면 제한적인 폴링을 함께 쓰는 식으로 설명하는 편입니다.

## 장점

- 동기/비동기 통신 방식을 비교해 설명하기 좋다.
- 외부 연동 경험이 없어도 실무 감각을 보여 주기 좋다.
- 멱등성, 재시도, 보안 검증 같은 후속 질문으로 자연스럽게 이어진다.

## 단점

- 폴링을 무조건 나쁜 방식처럼 말하면 답변이 단순해진다.
- 웹훅의 보안과 중복 처리 이슈를 빼면 답변 깊이가 얕아진다.
- 사용 시나리오 없이 개념만 비교하면 실제 판단 기준이 안 보일 수 있다.

## 주의사항 / 실무 팁

- 면접에서는 `누가 먼저 통신을 시작하느냐`를 한 문장으로 정리해 두면 좋다.
- 웹훅은 `서명 검증`과 `멱등 처리`를 같이 말해야 안정적이다.
- 폴링은 `주기`를 어떻게 잡을지까지 언급하면 실무적으로 들린다.
- 결제, 배달 상태, CI 완료 알림처럼 익숙한 사례를 하나 붙이면 답변이 훨씬 자연스럽다.
