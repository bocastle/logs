# 클래스풀(Classful) IP 주소 체계 정리

## 1) 배경: IPv4 주소 기본

- **IPv4 주소**는 32비트(4바이트)로 구성됩니다.
- 8비트(1바이트) 단위로 끊어 표기하며, 각 단위를 **옥텟(Octet)** 이라고 합니다.
- 범위: `0.0.0.0 ~ 255.255.255.255`

---

## 2) 클래스풀 주소 체계(Classful Addressing)란?

클래스풀 주소 체계는 IPv4 주소를 **고정된 네트워크/호스트 비트 길이**로 나누어, 주소를 **A/B/C(그리고 D/E)** 클래스처럼 규격화해 관리하는 방식입니다.

- 장점: 주소만 보면 “대략 어느 크기의 네트워크인지”를 빠르게 판단 가능
- 단점: 네트워크 크기가 **클래스별로 고정**되어 있어 **IP 낭비가 심함**

---

## 3) 클래스별 구조와 특징 (A, B, C)

### A 클래스

- **선행 비트(prefix):** `0`
- **네트워크/호스트:** 네트워크 1옥텟 / 호스트 3옥텟 (즉, /8)
- **네트워크 개수:** `2^7`
- **호스트 개수(네트워크당):** `2^24 - 2`
  - `-2`인 이유: 호스트 부분이 **모두 0(네트워크 주소)**, **모두 1(브로드캐스트 주소)** 는 예약

### B 클래스

- **선행 비트(prefix):** `10`
- **네트워크/호스트:** 네트워크 2옥텟 / 호스트 2옥텟 (즉, /16)
- **네트워크 개수:** `2^14`
- **호스트 개수(네트워크당):** `2^16 - 2`

### C 클래스

- **선행 비트(prefix):** `110`
- **네트워크/호스트:** 네트워크 3옥텟 / 호스트 1옥텟 (즉, /24)
- **네트워크 개수:** `2^21`
- **호스트 개수(네트워크당):** `2^8 - 2`

> 참고: D 클래스(멀티캐스트), E 클래스(실험용)는 일반적인 호스트/네트워크 할당 용도가 아닙니다.

---

## 4) 클래스풀의 한계: IP 주소 낭비

클래스풀은 네트워크 크기가 A/B/C로 **고정**되어 있어, 실제 필요한 호스트 수와 딱 맞지 않으면 낭비가 큽니다.

예) 어떤 조직이 호스트를 **255개** 필요로 한다면

- C 클래스(/24)는 최대 호스트가 `2^8 - 2 = 254`라서 **부족**
- 결국 B 클래스(/16)를 써야 하며, 그 결과 **호스트 주소 공간이 과도하게 커져 낭비**가 발생

이런 비효율을 해결하기 위해 **클래스리스(Classless) 주소 체계(CIDR)** 가 등장했습니다.

---

# 클래스리스(Classless) 주소 체계 정리

## 1) 클래스리스(Classless Addressing)란?

클래스리스 주소 체계는 A/B/C 같은 고정 클래스가 아니라, **서브넷 마스크(Subnet Mask)** 로 네트워크/호스트 경계를 정하는 방식입니다.

- 네트워크 비트: 연속된 `1`
- 호스트 비트: 연속된 `0`
- IP 주소와 서브넷 마스크를 **비트 AND 연산**하면 **네트워크 주소**를 얻습니다.

---

## 2) CIDR 표기법

서브넷 마스크를 다음과 같이 표기합니다.

- 형식: `IP주소/네트워크비트수`
- 예) `255.255.255.0`은 이진수로 1이 24개 → **/24**
  - `168.168.168.168/24`

---

## 3) 예시: 네트워크 주소 계산

- IP: `168.168.168.168`
- 서브넷 마스크: `255.255.255.0` (=/24)

AND 연산 결과 → 네트워크 주소:

- `168.168.168.0`

즉, 이 경우 호스트 영역은 마지막 1옥텟(8비트)이며, 사용 가능한 호스트 수는 `2^8 - 2 = 254`입니다.

---

## 4) 클래스리스의 장점

- 필요한 호스트 수에 맞게 **/20, /26** 처럼 유연하게 네트워크를 쪼갤 수 있어 **IP 낭비를 크게 줄임**
- 라우팅 테이블 집계(슈퍼네팅) 등 **라우팅 효율 개선**에도 유리
