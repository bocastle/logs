# 연속 메모리 할당 기법(Continuous Memory Allocation) 정리

## 핵심 요약

- **연속 메모리 할당**은 운영체제가 프로세스에 **연속적인 물리 메모리 공간**을 할당하는 방식입니다.
- 방식은 크게 **고정 크기 할당**과 **가변 크기 할당**으로 나뉘며, 각각 **내부 단편화/외부 단편화**와 밀접한 관련이 있습니다.
- 빈 공간이 여러 개일 때 배치 전략으로 **최초 적합(First Fit)**, **최적 적합(Best Fit)**, **최악 적합(Worst Fit)** 을 사용할 수 있습니다.

## 개요

연속 메모리 할당 기법(Continuous Memory Allocation)은 운영체제가 프로세스에 연속적인 메모리 공간을 할당하는 방법을 의미합니다. 연속 메모리 할당 기법을 사용하면, 하나의 프로세스는 메모리 주소 공간에서 연속적으로 존재하게 됩니다.

연속 메모리 할당 기법은 크게 **가변 크기 메모리 할당**과 **고정 크기 메모리 할당**이 존재합니다.

### 고정 크기 메모리 할당

- 물리적인 메모리 공간을 **고정된 크기**로 나누어 프로세스에 **고정된 크기만큼** 할당합니다.

### 가변 크기 메모리 할당

- 프로세스의 크기에 맞춰 **동적으로** 메모리를 할당합니다.

## 외부 단편화와 내부 단편화

### 외부 단편화(External Fragmentation)

외부 단편화는 프로세스를 할당하기 어려울 만큼 **작은 메모리 공간들이 흩어져** 있어서 메모리가 낭비되는 현상입니다.

- 예: 프로세스가 300MB의 연속 공간을 요구한다고 가정했을 때
  - 흩어져 있는 빈 공간들의 합은 300MB가 넘지만
  - **연속적인 300MB**가 존재하지 않아 메모리를 할당하지 못하는 상황이 발생할 수 있습니다.

### 내부 단편화(Internal Fragmentation)

내부 단편화는 메모리를 할당할 때 프로세스가 필요한 양보다 **더 큰 메모리가 할당**되어, 그 차이만큼 메모리가 낭비되는 현상입니다.

- 예: 프로세스가 300MB를 사용하지만 500MB가 할당되면 200MB가 낭비됩니다.

### 단편화 관점에서의 비교

- **고정 크기 할당**: 고정된 크기만큼 할당하므로 **내부 단편화가 발생**할 수 있습니다.
- **가변 크기 할당**: 필요한 크기만큼 할당하므로 **내부 단편화는 발생하지 않지만**, 할당/해제를 반복하면 **외부 단편화가 발생**할 수 있습니다.

## 빈 공간이 여러 개일 때 배치 전략

연속 메모리 할당 기법에서 사용할 수 있는 대표적인 방법으로 **최초 적합**, **최적 적합**, **최악 적합** 방식이 존재합니다.

### 최초 적합(First Fit)

- 운영체제가 메모리 내의 빈 공간을 **순서대로 검색**하고, **최초로 발견된 공간**에 프로세스를 배치합니다.

### 최적 적합(Best Fit)

- 운영체제가 메모리 내의 빈 공간을 **모두 검색**하고, 적재될 수 있는 **가장 작은 공간**에 프로세스를 배치합니다.

### 최악 적합(Worst Fit)

- 운영체제가 메모리 내의 빈 공간을 **모두 검색**하고, 적재될 수 있는 **가장 큰 공간**에 프로세스를 배치합니다.

## 장점

- 하나의 프로세스가 메모리 주소 공간에서 연속적으로 존재하므로(연속 공간 할당) 메모리 상의 배치가 단순합니다.

## 단점

- 할당/해제를 반복할수록 단편화(특히 외부 단편화)가 문제가 될 수 있으며, 필요한 크기의 연속 공간을 확보하지 못해 할당이 실패할 수 있습니다.

## 주의사항 및 실무 팁

- 고정 크기 할당은 내부 단편화가 발생할 수 있으므로, **고정 블록 크기**를 어떻게 잡는지가 낭비율에 큰 영향을 줍니다.
- 가변 크기 할당은 내부 단편화는 줄일 수 있지만, 장기적으로 외부 단편화가 누적될 수 있으니 **할당/해제 패턴**에 따라 메모리 활용률이 달라질 수 있습니다.
- 배치 전략(First/Best/Worst Fit)은 상황에 따라 성능과 단편화 정도가 달라질 수 있으므로, 목표(검색 비용 vs 단편화 감소)를 기준으로 선택합니다.
